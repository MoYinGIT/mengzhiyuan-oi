排序分治线

从L4启动，L5深化——理解"分治"思想，掌握归并排序和快速排序。 排序是手段，分治是目的。

排序教学最大的误区，是只教排序，不教分治。

归并排序和快速排序的价值，不在于"排序"这个结果，而在于"分治"这个过程。 分治思想是算法设计的核心范式，必须在L4-L5建立起来。

为什么排序要从"为什么排"引入？

错误做法

直接教代码：

void qsort(int l, int r) {
  if (l >= r) return;
  int mid = ...
  // ...
}

学生背下来，多做几遍就会了。

正确做法

先问：为什么要排序？

  • 为了快速查找（二分）
  • 为了去重（相邻比较）
  • 为了贪心选择（排序后取最大）

再问：如何高效排序？

引出分治思想。

排序分治线的发展阶段

L4
基础排序期 启动
核心内容：
  • 选择排序、冒泡排序（理解排序本质）
  • 插入排序（近乎有序的优化）
  • 排序的应用（查找、去重、贪心）
思维目标：理解"为什么排序"比"怎么排序"更重要
L5
分治排序期 深化
核心内容：
  • 归并排序（稳定的分治）
  • 快速排序（高效的分治）
  • 分治思想抽象：分解→解决→合并
  • 逆序对计数（分治应用）
思维目标：建立分治思想，理解递归与分治的关系

分治思想的核心

分治三步法

分解：把大问题拆成若干个小问题
解决：递归解决每个小问题
合并：把小问题的解合并成大问题的解

关键洞察

归并排序和快速排序的区别在于"分解"的方式：归并排序是"先分后治"（稳定但需额外空间），快速排序是"先治后分"（不稳定但原地排序）。理解这个区别，就理解了分治的灵活性。

常见问题

Q1：学生需要先掌握递归再学分治吗？

是的。分治依赖递归实现。这也是为什么排序分治线在L4启动（此时递归线已经进入L3末-L4阶段）。两条主线在此处交汇，相互支撑。

Q2：实际编程中需要手写排序吗？

通常不需要，sort()函数足够。但学习排序的价值在于理解分治思想。就像学微积分不是为了手算积分表，而是为了理解变化的本质。
