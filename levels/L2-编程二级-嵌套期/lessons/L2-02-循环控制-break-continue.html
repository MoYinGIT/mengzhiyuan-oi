L2-02 循环控制 - break / continue

6课时（270分钟） L2核心

教学目标

知识目标

  • 理解 break 和 continue 的作用和区别
  • 掌握在多重循环中使用 break 的规则
  • 理解循环控制对程序效率的影响

技能目标

  • 能正确使用 break 提前结束循环
  • 能正确使用 continue 跳过当前迭代
  • 能用循环控制优化枚举算法

break vs continue

break
立即终止整个循环，跳出循环体，执行循环后面的代码。

比喻：比赛途中退赛，彻底结束。
continue
跳过本次循环的剩余代码，直接进入下一次循环判断。

比喻：比赛途中跳过这一轮，下一轮继续。

代码对比

C++ 
// break 示例：找到第一个能被7整除的数
for (int i = 1; i <= 100; i++) {
    if (i % 7 == 0) {
        cout << "找到了：" << i << endl;
        break;  // 找到就停止，不再继续查找
    }
}

// continue 示例：输出1-10中所有奇数
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) {
        continue;  // 偶数跳过，不输出
    }
    cout << i << " ";  // 只输出奇数
}

多重循环中的 break

重要规则

break 只跳出当前所在的那一层循环，不会跳出所有外层循环！

C++ 
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        if (j == 2) {
            break;  // 只跳出内层 j 循环
        }
        cout << i << "," << j << " ";
    }
    cout << endl;
}

// 输出：
// 1,1 
// 2,1 
// 3,1

IOI教练视角

陷阱：continue 后忘记更新变量

如果在 continue 之前有更新循环变量的代码，被 continue 跳过后可能导致死循环。比如 while 循环中，continue 跳过了 i++。

优化思维：break 的价值在于"找到答案后立刻停止"，避免不必要的计算。这是竞赛中常用的优化技巧——能提前结束就绝不多做一步。

练习设计

基础练习

  • 输入 n，判断 n 是否为质数（用 break 优化）
  • 输出 1-100 中所有能被 3 整除但不能被 5 整除的数（用 continue）
  • 找到 100 以内最大的质数

提高练习

  • 输入 n 个数，找到第一个大于 100 的数并输出其位置
  • 打印 1-100，跳过所有包含数字 7 的数

  • 知道 break 只跳出当前层循环
  • 能用 break 优化质数判断等算法
  • 不会因 continue 导致死循环
