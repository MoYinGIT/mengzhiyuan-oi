枚举算法的核心思想

枚举算法的本质就是：把所有可能的答案一个一个试过去，符合条件的就留下。

听起来很笨？但很多问题，只要枚举范围不大，这就是最简单、最可靠的方法。而且，很多高级算法的本质也是优化过的枚举。

枚举三步法

确定范围

明确需要枚举哪些对象，范围是什么。

关键 范围不能太大，否则超时

逐一尝试

用循环遍历范围内的每一个候选答案。

关键 确保不遗漏、不重复

条件筛选

对每个候选答案进行验证，符合就输出或记录。

关键 验证条件要写对

经典案例

案例1：百钱买百鸡

公鸡5钱一只，母鸡3钱一只，小鸡1钱三只。用100钱买100只鸡，问公鸡、母鸡、小鸡各多少只？

C++ 
// 枚举公鸡和母鸡的数量，小鸡数量由总数确定
for (int x = 0; x <= 20; x++) {        // 公鸡最多20只
    for (int y = 0; y <= 33; y++) {    // 母鸡最多33只
        int z = 100 - x - y;           // 小鸡数量
        if (z % 3 == 0 && 5 * x + 3 * y + z / 3 == 100) {
            cout << x << " " << y << " " << z << endl;
        }
    }
}

案例2：找出1-1000中的所有质数

C++ 
for (int i = 2; i <= 1000; i++) {
    bool isPrime = true;
    for (int j = 2; j * j <= i; j++) {
        if (i % j == 0) {
            isPrime = false;
            break;  // 找到因子就停止
        }
    }
    if (isPrime) {
        cout << i << " ";
    }
}

优化思维：判断质数时，只需枚举到 sqrt(i) 即可。这是枚举优化的典型例子——缩小枚举范围。

IOI教练视角

陷阱：枚举范围过大

初学枚举最容易犯的错误就是范围设得太大，导致程序运行缓慢甚至超时。枚举前一定要先估算范围，尽量缩小。

陷阱：遗漏边界

枚举范围的起止点要仔细考虑。比如"1到100"应该写成 i = 1; i <= 100，不要漏掉等于的情况。

练习设计

基础练习

  • 输入 n，找出 1-n 中所有能被 3 或 5 整除的数
  • 输入 n，找出第 n 个质数
  • 输入一个正整数，找出它的所有因数

提高练习

  • 完美数：找出 1-1000 中的所有完美数（等于其真因数之和的数，如 6 = 1+2+3）
  • 水仙花数：找出所有三位水仙花数
  • 鸡兔同笼：已知头数和脚数，求鸡和兔各多少只
