状态表示与哈希

状态编码、去重、set/map应用、记忆化搜索入门

8课时
约2周
难度

本课三维目标

知识 状态编码方法、哈希去重、set/map基本用法、记忆化搜索概念
技能 能将复杂状态编码为字符串/整数，并用set去重
思维 建立"状态是唯一标识"的抽象思维，为图论和DP打基础

1. 什么是状态？

在搜索问题中，状态就是描述当前局面的全部信息。 比如迷宫问题中，当前坐标 (x, y) 就是一个状态；八数码问题中，棋盘上3×3的数字排列就是一个状态。

如果两个状态的描述完全相同，那么从这两个状态出发，后续的所有可能性也完全相同。 因此，如果我们已经访问过某个状态，就不需要再次访问。

2. 状态编码：把状态变成可比较的对象

计算机要判断"这个状态是否访问过"，需要先把状态编码成一种统一的格式，比如字符串或整数。

示例1：迷宫中的坐标

坐标编码 
// 方法1：整数编码
int encode(int x, int y) {
    return x * 1000 + y;  // 假设坐标都小于1000
}

// 方法2：字符串编码
string encode(int x, int y) {
    return to_string(x) + "," + to_string(y);
}

示例2：3×3 棋盘状态（八数码）

棋盘状态编码 
int board[3][3] = {{1,2,3},{4,5,6},{7,8,0}};

// 把棋盘展平成一个9位数字
long long encode() {
    long long code = 0;
    for (int i = 0; i < 3; i++)
        for (int j = 0; j < 3; j++)
            code = code * 10 + board[i][j];
    return code;  // 例如 123456780
}

3. 用 set 进行状态去重

C++的 set 和 unordered_set 可以高效地存储和查询已访问的状态。

set去重模板 
#include <unordered_set>
using namespace std;

unordered_set<string> visited;  // 存储已访问的状态

void bfs(string start) {
    queue<string> q;
    q.push(start);
    visited.insert(start);
    
    while (!q.empty()) {
        string state = q.front();
        q.pop();
        
        // 生成所有邻接状态
        for (string next : getNeighbors(state)) {
            if (visited.find(next) == visited.end()) {
                visited.insert(next);
                q.push(next);
            }
        }
    }
}

注意：unordered_set 的查询平均是 O(1)，比 set 的 O(log n) 更快。 但 unordered_set 对自定义类型需要写哈希函数。对于字符串和整数，直接使用即可。

4. 记忆化搜索入门

记忆化搜索是"搜索 + 动态规划"的结合。它的核心思想是： 把已经计算过的状态的答案存起来，下次遇到同样的状态时直接查表。

记忆化搜索示例：斐波那契 
long long memo[100];

long long fib(int n) {
    if (n <= 1) return 1;
    if (memo[n] != 0) return memo[n];  // 查表！
    
    memo[n] = fib(n - 1) + fib(n - 2); // 计算并存储
    return memo[n];
}

记忆化搜索把原本指数级的递归复杂度降到了线性。 它的本质是"用空间换时间"，和数组DP线的思想一脉相承。

5. map的应用：状态到答案的映射

当状态不能用简单的数组下标表示时（比如字符串状态），可以用 map 或 unordered_map 来存储"状态→答案"的映射。

map存储状态答案 
unordered_map<string, int> dist;  // 状态 -> 最短距离

void bfs(string start) {
    queue<string> q;
    q.push(start);
    dist[start] = 0;
    
    while (!q.empty()) {
        string state = q.front(); q.pop();
        
        for (string next : getNeighbors(state)) {
            if (dist.find(next) == dist.end()) {
                dist[next] = dist[state] + 1;
                q.push(next);
            }
        }
    }
}

IOI教练常见教学陷阱

陷阱1：状态编码不唯一

如果两种不同局面生成了相同的编码，去重就会出错。 设计编码时要确保它是一一对应的（双射）。

陷阱2：过早使用复杂哈希

L5阶段不需要教MD5、SHA等密码学哈希。用整数编码、字符串拼接或STL内置哈希就够了。

陷阱3：混淆visited和dist

在BFS中，可以用dist的默认值（如-1）表示未访问，不需要单独维护visited。 但在状态空间搜索中，可能需要用set专门做去重。
