回溯法

全排列、N皇后、子集生成、状态恢复

8课时
约2周
难度

本课三维目标

知识 回溯法定义、选择-递归-撤销三步、状态恢复机制
技能 能用回溯法解决全排列、N皇后、子集生成等经典问题
思维 建立"尝试-失败-回退-再尝试"的系统性枚举思维

回溯法可以看作是DFS在组合/排列/选择问题上的特化应用。 它的核心只有一句话：做选择 → 递归深入 → 撤销选择。 很多学生学不会回溯，不是因为代码难，而是因为没理解"为什么要撤销"。 这一课，我们要把这个机制讲透。

1. 回溯法的核心思想

回溯法是一种系统性枚举策略：在搜索过程中，当我们发现当前选择无法通向有效解时， 就回退到上一步，撤销刚才的选择，然后尝试其他选择。

你可以把回溯想象成走迷宫时用粉笔在墙上做标记：走进一个岔路，发现是死胡同， 擦掉标记，退回岔路口，尝试另一条路。

graph TD A[开始] --> B[做选择1] B --> C[递归深入] C --> D[发现死胡同] D --> E[撤销选择1] E --> F[做选择2] F --> G[递归深入] G --> H[找到解]

2. 全排列：回溯法的经典入门

问题：给定数字 1, 2, 3，输出它们的所有排列。

全排列回溯代码 
const int MAXN = 15;
int n;
int path[MAXN];      // 当前路径
bool used[MAXN];     // 数字是否已被使用

void backtrack(int step) {
    if (step == n) {
        // 已经选了n个数字，输出排列
        for (int i = 0; i < n; i++) {
            cout << path[i] << " ";
        }
        cout << endl;
        return;
    }
    
    for (int i = 1; i <= n; i++) {
        if (!used[i]) {
            // 做选择
            path[step] = i;
            used[i] = true;
            
            // 递归深入
            backtrack(step + 1);
            
            // 撤销选择（关键！）
            used[i] = false;
            path[step] = 0; // 可选
        }
    }
}

注意代码结构：做选择 → 递归 → 撤销选择。这三步是回溯法的固定舞步。

3. 状态恢复：为什么必须撤销？

如果不撤销 used[i]，当循环回到上一层时， i 仍然被标记为已使用，导致后续排列无法正确生成。

撤销的本质是把共享状态恢复到尝试之前，让下一次循环能在一个"干净"的环境中做新的选择。

4. N皇后问题

问题：在 n×n 的棋盘上放置 n 个皇后，使它们互不攻击（不在同一行、同一列、同一对角线）。 求所有可行的摆法。

N皇后回溯代码 
const int MAXN = 15;
int n, ans;
int col[MAXN];  // col[row] = 皇后放在第row行第col列

bool isValid(int row, int c) {
    for (int i = 0; i < row; i++) {
        // 同列 或 同对角线
        if (col[i] == c || abs(col[i] - c) == abs(i - row)) {
            return false;
        }
    }
    return true;
}

void backtrack(int row) {
    if (row == n) {
        ans++;
        return;
    }
    
    for (int c = 0; c < n; c++) {
        if (isValid(row, c)) {
            col[row] = c;      // 做选择
            backtrack(row + 1); // 递归
            col[row] = -1;     // 撤销选择
        }
    }
}

N皇后是回溯法的试金石。它要求学生不仅要会写递归，还要会设计合法性检查函数。

5. 子集生成

问题：给定一个集合，输出它的所有子集。

子集生成回溯代码 
int a[] = {1, 2, 3};
vector<int> subset;

void backtrack(int index) {
    if (index == 3) {
        // 输出当前子集
        for (int x : subset) cout << x << " ";
        cout << endl;
        return;
    }
    
    // 选择 a[index]
    subset.push_back(a[index]);
    backtrack(index + 1);
    subset.pop_back();  // 撤销
    
    // 不选择 a[index]
    backtrack(index + 1);
}

每个元素都有"选"或"不选"两种决策，形成一棵二叉决策树。回溯法会遍历这棵树的所有叶子节点。

IOI教练常见教学陷阱

陷阱1：忘记撤销选择

这是回溯法初学者的头号大敌。没有撤销，状态就会被污染，后续搜索会基于错误的状态进行。

陷阱2：把回溯法和DFS割裂开

回溯法就是DFS的一种应用形式。它们的底层机制完全一样：递归+状态维护+回溯。 告诉学生这一点，有助于知识迁移。

陷阱3：一上来就讲N皇后

N皇后对初学者较难。建议教学顺序：全排列 → 子集生成 → N皇后。 由简入繁，建立信心。
