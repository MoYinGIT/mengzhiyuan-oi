搜索剪枝

可行性剪枝、最优性剪枝、剪枝的艺术与边界

8课时
约2周
难度

本课三维目标

知识 剪枝的定义、可行性剪枝、最优性剪枝、剪枝的边界条件
技能 能在DFS/BFS/回溯中识别并加入有效的剪枝条件
思维 建立"提前终止无效搜索"的优化意识

搜索的本质是枚举。但枚举所有可能性在很多问题中会超时（TLE）。 剪枝就是在搜索树还没长到叶子的时候，就判断这条路不可能通向答案， 然后果断砍掉它。一个优秀的OI选手，和一个普通选手的区别，往往就在于剪枝的艺术。

1. 为什么要剪枝？

假设我们要在1~20中选出若干个数，使它们的和等于20。如果不剪枝，回溯法会枚举所有 2^20 ≈ 100万 个子集。

但如果我们发现当前已选数字的和已经超过20了，就可以立即停止这条路径的搜索—— 因为再往下加只会更大。这个简单的判断，可能砍掉搜索树的半壁江山。

2. 可行性剪枝

可行性剪枝：当当前状态已经违反问题的约束条件时，直接返回，不再继续搜索。

和为target的子集问题 
void dfs(int index, int currentSum) {
    // 可行性剪枝：和已经超过target
    if (currentSum > target) return;
    
    // 可行性剪枝：就算把剩下所有数都加上，也凑不够target
    if (currentSum + remainingSum[index] < target) return;
    
    if (index == n) {
        if (currentSum == target) ans++;
        return;
    }
    
    // 选 a[index]
    dfs(index + 1, currentSum + a[index]);
    
    // 不选 a[index]
    dfs(index + 1, currentSum);
}

上面的代码展示了两种可行性剪枝：

  • 上界剪枝：当前和已经超过target，不可能再满足条件
  • 下界剪枝：即使加上所有剩余数字，和仍然小于target，也不可能满足条件

3. 最优性剪枝

最优性剪枝：当当前状态已经不可能比已知最优解更优时，直接返回。 它常用于求最大值/最小值的问题。

最优性剪枝示例 
int best = INF;  // 当前找到的最短路径长度

void dfs(int x, int y, int step) {
    // 最优性剪枝：已经比已知最优解差了
    if (step >= best) return;
    
    if (x == ex && y == ey) {
        best = step;  // 更新最优解
        return;
    }
    
    // ... 四个方向搜索 ...
}

最优性剪枝的威力在于：越早找到一个好的解，就越能砍掉更多的分支。 因此很多题目会先写一个贪心算法求一个"不那么差"的初始解，再用搜索优化。

4. 常见剪枝技巧汇总

剪枝类型 适用场景 判断依据
越界剪枝 网格搜索 坐标超出边界
重复剪枝 排列/组合 当前元素与上一个相同且上一个未使用
和/积剪枝 子集、背包类 当前和（积）已超过目标，或加上剩余也不够
顺序剪枝 组合选择 规定选择的顺序，避免重复状态

5. 剪枝的注意事项

剪枝不能剪错！ 一个错误的剪枝条件，可能把正确答案所在的子树一起砍掉， 导致程序输出错误答案（WA）。

  • 剪枝条件必须是充分条件：满足这个条件，一定不可能有解
  • 不要过早优化：先把裸搜索写出来，再考虑加剪枝
  • 剪枝条件本身不要写得太复杂，避免引入新的Bug和额外开销

IOI教练常见教学陷阱

陷阱1：还没写出裸搜索就急着剪枝

很多学生一听剪枝很酷，就开始在代码里到处加if。 结果是逻辑混乱，剪枝条件写错，程序WA。正确顺序：先写裸搜索，确认正确，再加剪枝。

陷阱2：剪枝条件过于保守

有些学生加了剪枝，但只砍掉很少的分支，对效率提升微乎其微。 好的剪枝应该能砍掉至少一个数量级的搜索空间。

陷阱3：认为剪枝是万能的

有些问题（如某些NP-hard问题）即使加了剪枝，复杂度仍然是指数级的。 剪枝能优化常数或降低平均复杂度，但不能改变问题的本质难度。

课后练习题

1

和为target的子集计数

入门

给定 n 个正整数，求和为 target 的子集数量。要求加入上下界剪枝。

2

迷宫最短路径（DFS+剪枝）

重点

用DFS求迷宫最短路径，加入最优性剪枝（step ≥ best时返回）。

3

N皇后优化

重点

在标准N皇后代码中加入对称性剪枝：利用棋盘对称性减少搜索量。

4

24点游戏

挑战

给定4个数字，通过加减乘除和括号运算，判断是否能得到24。用搜索+剪枝解决。
