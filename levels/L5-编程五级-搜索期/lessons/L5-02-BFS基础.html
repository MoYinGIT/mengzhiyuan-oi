BFS基础

广度优先搜索、队列实现、最短路径、层序遍历

8课时
约2周

1. 什么是BFS？

BFS（Breadth First Search，广度优先搜索）是一种搜索策略： 从起点出发，先访问所有与起点距离为1的邻居，再访问距离为2的邻居，依此类推，像水波一样一圈圈向外扩散。

BFS的最大特点是：在无权图中，它第一次到达某个节点时，走过的路径一定是最短的。 因此BFS是求解最短路径问题的首选算法。

2. BFS的核心代码模板

BFS使用队列（queue）来实现。队列的特点是"先进先出"（FIFO），这保证了先被访问的节点会先处理它的邻居。

BFS标准模板 
#include <queue>
using namespace std;

const int MAXN = 105;
int n, m;
char maze[MAXN][MAXN];
int dist[MAXN][MAXN];  // 记录到每个点的最短距离

int dx[4] = {-1, 1, 0, 0};
int dy[4] = {0, 0, -1, 1};

int bfs(int sx, int sy, int ex, int ey) {
    queue<pair<int, int>> q;
    
    // 初始化距离数组为-1，表示未访问
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            dist[i][j] = -1;
    
    dist[sx][sy] = 0;
    q.push({sx, sy});
    
    while (!q.empty()) {
        auto [x, y] = q.front();  // C++17 结构化绑定
        q.pop();
        
        for (int i = 0; i < 4; i++) {
            int nx = x + dx[i];
            int ny = y + dy[i];
            
            if (nx >= 0 && nx < n && ny >= 0 && ny < m 
                && maze[nx][ny] == '.' 
                && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;
                q.push({nx, ny});
            }
        }
    }
    
    return dist[ex][ey];  // 返回-1表示不可达
}

核心口诀：起点入队 → 标记距离 → 出队扩展 → 邻居入队。 dist数组在这里同时起到了visited的作用：dist=-1表示未访问。

3. BFS vs DFS 对比

特性 DFS BFS
数据结构 递归（系统栈） 队列（手动维护）
搜索顺序 一条路走到黑 一圈一圈扩散
最短路径 不一定最短 无权图一定最短
适用场景 连通块、回溯、枚举所有路径 最短路径、层序遍历、最小步数

5. 多源BFS（拓展）

有时候问题有多个起点，比如"求地图上每个格子到最近火源的距离"。 这时可以把所有起点同时入队，这就是多源BFS。

多源BFS核心代码 
// 把所有起点同时入队
for (每个起点 (sx, sy)) {
    dist[sx][sy] = 0;
    q.push({sx, sy});
}

while (!q.empty()) {
    // ... 正常BFS扩展 ...
}

多源BFS的思想在后续图论（L7）中会大量用到，比如SPFA算法的前身就是BFS思想的拓展。

IOI教练常见教学陷阱

陷阱1：用DFS求最短路径

初学者最容易犯的错误。在无权网格图中，求最短路径必须用BFS。 DFS虽然也能找到路径，但不保证最短。

陷阱2：dist和visited混用

建议BFS中只用dist数组，dist=-1表示未访问，dist≥0表示已访问且记录了距离。 不要同时维护visited和dist两个数组，容易混乱。

陷阱3：队列元素出队后才更新邻居

有些学生习惯先把邻居入队，等出队了再标记dist。 这会导致同一个节点被多次入队。正确做法是入队前就标记dist。
