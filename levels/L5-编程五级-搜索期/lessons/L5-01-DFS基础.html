DFS基础

深度优先搜索、递归实现、迷宫探索、连通块计数

8课时
约2周
难度

本课三维目标

知识 DFS定义、访问标记、四方向移动、递归回溯
技能 能独立写出标准DFS模板，解决迷宫可达性和连通块问题

1. 什么是DFS？

DFS（Depth First Search，深度优先搜索）是一种搜索策略： 从起点出发，沿着一条路径一直走到底，如果走不通了，就退回上一个分岔口，换一条路继续走。

你可以把DFS想象成一个探险家：他进入一个洞穴后，总是选择最左边的通道往前走， 直到死胡同才原路返回，然后在最近的岔路口探索另一条通道。

2. DFS的核心代码模板

在网格迷宫问题中，DFS的标配是：访问标记数组 + 四方向移动 + 递归。

DFS标准模板 
const int MAXN = 105;
int n, m;
char maze[MAXN][MAXN];
bool visited[MAXN][MAXN];

int dx[4] = {-1, 1, 0, 0};
int dy[4] = {0, 0, -1, 1};

void dfs(int x, int y) {
    visited[x][y] = true;  // 标记为已访问
    
    for (int i = 0; i < 4; i++) {
        int nx = x + dx[i];
        int ny = y + dy[i];
        
        // 检查边界、是否可走、是否访问过
        if (nx >= 0 && nx < n && ny >= 0 && ny < m 
            && maze[nx][ny] == '.' 
            && !visited[nx][ny]) {
            dfs(nx, ny);
        }
    }
}

核心口诀：标记访问 → 四个方向 → 检查合法 → 递归深入。 这四步是99%网格DFS的固定写法。

3. 为什么需要visited数组？

如果没有 visited 标记，DFS会在两个相邻的格子之间反复横跳， 造成无限循环。

visited 数组的作用就是"走过的路不再走"。 这是DFS和BFS都必须遵守的铁律。

4. 经典应用：连通块计数

问题：给定一个地图，'#' 是陆地，'.' 是海洋。 相邻（上下左右）的陆地属于同一个岛屿，问地图上有多少个岛屿？

连通块计数 
int countIslands() {
    int count = 0;
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            if (maze[i][j] == '#' && !visited[i][j]) {
                dfs(i, j);  // 发现一个未访问的陆地，开始DFS
                count++;    // 每调用一次dfs，就是一个新岛屿
            }
        }
    }
    return count;
}

核心思想：外层循环扫描每个格子，遇到未访问的陆地就"派出"一次DFS， 把与之相连的所有陆地都标记为已访问。派出几次DFS，就有几个岛屿。

IOI教练常见教学陷阱

陷阱1：忘记标记visited

这是DFS初学者最高频的Bug。没有visited会导致死循环或栈溢出。 每次写dfs，第一件事就是写 visited[x][y] = true;。

陷阱2：边界检查写错

常见错误是 nx > 0 而不是 nx >= 0， 或者把 n 和 m 搞反。 建议用标准模板，不要每次都重新写边界条件。

陷阱3：认为DFS一定能找到最短路径

DFS是"深度优先"，它找到的路径通常不是最短的。 求最短路径应该使用BFS。这是一个极其常见的误解。

1

迷宫可达性

入门

给定迷宫、起点和终点，判断是否存在从起点到终点的路径。

2

岛屿数量

重点

给定 n×m 地图，'1'表示陆地，'0'表示海洋，统计岛屿数量。

3

迷宫最长路径

重点

给定迷宫，求从起点到终点的最长简单路径长度（每个格子最多走一次）。

4

八方向连通块

挑战

岛屿计数变体：除了上下左右，对角线方向也算相邻。需要8个方向搜索。
