变量作用域

局部变量、全局变量、作用域规则、生命周期

8课时
约2周
难度

本课三维目标

知识 局部变量、全局变量、作用域、生命周期、命名冲突
技能 能合理选择局部/全局变量，避免命名冲突和作用域错误
思维 理解"变量的可见范围"与"存活时间"是两个不同概念

1. 局部变量 vs 全局变量

作用域对比 
int global = 100; // 全局变量：所有函数都能访问

void foo() {
    int local = 10; // 局部变量：只能在 foo 内访问
    cout << global << endl; // ✅ 可以访问全局变量
    cout << local << endl;  // ✅ 可以访问自己的局部变量
}

int main() {
    cout << global << endl; // ✅ 可以访问全局变量
    // cout << local << endl; // ❌ 编译错误！local 在 foo 里定义
    return 0;
}
特性 局部变量 全局变量
定义位置 函数内部 函数外部
可见范围 只在定义它的函数/块内 整个程序的所有函数
生命周期 函数执行时存在，结束后销毁 程序开始时就存在，结束时销毁
初始值 不确定（垃圾值） 自动初始化为 0

2. 命名冲突：局部优先

如果局部变量和全局变量同名，函数内部优先使用局部变量。 这叫做"局部遮蔽"（shadowing）。

同名变量的优先级 
int x = 100; // 全局变量

void test() {
    int x = 10; // 局部变量
    cout << x << endl; // 输出 10（局部优先）
}

int main() {
    test();
    cout << x << endl; // 输出 100（全局变量）
    return 0;
}

建议：尽量避免局部变量和全局变量同名。这会让代码非常难读，也容易出现意想不到的Bug。

3. 块级作用域

在C++中，花括号 {} 不仅用于函数体，还定义了一个代码块。 在代码块内定义的变量，只能在该代码块内访问。

块级作用域示例 
int main() {
    int a = 10;
    if (a > 5) {
        int b = 20; // b 只在 if 块内有效
        cout << b << endl; // ✅ 合法
    }
    // cout << b << endl; // ❌ 编译错误！b 已经"死"了
    
    for (int i = 0; i < 5; i++) {
        // i 只在 for 循环内有效
    }
    // cout << i << endl; // ❌ 编译错误（某些旧编译器可能允许）
    
    return 0;
}

4. 全局数组：竞赛中的标配

在竞赛编程中，大数组通常定义为全局变量。原因有三：

  1. 全局数组空间更大，不会因为栈溢出而崩溃
  2. 全局数组自动初始化为0，省去手动清零
  3. 所有函数都可以直接访问，方便多函数协作
竞赛标准写法 
const int MAXN = 100005;
int a[MAXN];      // 全局数组，自动清零
long long pre[MAXN]; // 前缀和数组

void readData(int n) {
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }
}

int solve(int n) {
    // 直接访问全局数组 a
    int sum = 0;
    for (int i = 0; i < n; i++) {
        sum += a[i];
    }
    return sum;
}

5. 全局变量的滥用与节制

全局变量虽然方便，但也有明显缺点：

  • 可读性下降：任何函数都可能修改它，追踪数据流很困难
  • 可维护性差：程序变大后，全局变量互相干扰
  • 并发问题：多线程环境下全局变量是重灾区

L4阶段的建议：数组和常量可以全局定义；普通的计数器、中间结果尽量用局部变量。 养成"能用局部就不用全局"的好习惯。

IOI教练常见教学陷阱

陷阱1：把大数组定义在函数内部

很多学生习惯把所有变量都定义在main里。竞赛中10^5以上的数组必须放全局， 否则栈溢出会导致神秘的运行时错误。

陷阱2：全局变量不初始化就使用

虽然全局变量会自动初始化为0，但依赖这一点而不显式初始化， 会让代码可读性变差。重要变量建议显式赋值。

陷阱3：局部变量和循环变量同名

外层循环用 i，内层循环也用 i，会导致内层覆盖外层。 虽然合法，但极容易出错。嵌套循环建议用 i, j, k 区分。

课后练习题

1

预测输出

入门

给出多段包含全局变量和局部变量冲突的代码，让学生预测输出结果。

2

作用域Bug修复

重点

给定一段因变量作用域问题导致编译错误的程序，要求学生修复。

3

全局vs局部数组

重点

比较把同样大小的数组定义在main内和main外的区别，解释为什么竞赛常用全局数组。

4

模块化改造

挑战

将一个滥用全局变量的程序，改造为尽量使用局部变量和参数传递的版本。
