分治初探

分治思想、归并排序思想、二分查找、问题拆分

8课时
约2周
难度

本课三维目标

知识 分治思想、二分查找原理、归并排序的基本流程
技能 能独立写出二分查找代码，理解归并排序的拆分与合并
思维 建立"大问题拆成小问题，分别解决再合并"的分治思维

分治是算法设计中最优美的思想之一。从二分查找到归并排序，从快速幂到线段树， 背后都是同一个逻辑："如果我能解决一半，就能解决全部"。 L4阶段，我们不求学生能手写归并排序代码，但求他们理解"拆分-解决-合并"这个循环。

🚫 L4排序铁律（暴力肌肉化）：本阶段禁止调用任何现成排序函数（包括 sort()）。 写选择排序、冒泡排序时，必须在草稿纸上画出每一轮交换后的数组状态。 把"写代码"转化为"用代码模拟手工操作"，直到手指形成"找最小值→交换到末尾"的机械反射。 sort() 是L5的奖励，不是L4的拐杖。

▲ 草稿纸标准：每轮交换后必须画出当前数组状态（红色=本轮交换位置）

1. 分治思想：Divide and Conquer

分治算法的三个步骤：

  1. Divide（拆分）：把大问题拆成若干个规模更小的子问题
  2. Conquer（解决）：递归地解决每个子问题
  3. Combine（合并）：把子问题的结果合并成原问题的答案
graph TD A[原问题] --> B[子问题1] A --> C[子问题2] B --> D[更小子问题] B --> E[更小子问题] C --> F[更小子问题] C --> G[更小子问题] D --> H[直接求解] E --> I[直接求解] F --> J[直接求解] G --> K[直接求解] H --> L[合并结果] I --> L J --> M[合并结果] K --> M L --> N[最终答案] M --> N

2. 二分查找：分治的最简形态

问题：在有序数组中查找某个数是否存在。

朴素做法：从头扫到尾，O(n)。
分治做法：先看中间元素，如果目标更小，就在左半部分继续找；如果更大，就在右半部分继续找。 复杂度：O(log n)。

二分查找模板 
int binarySearch(int a[], int n, int target) {
    int left = 0, right = n - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2; // 防止溢出
        if (a[mid] == target) return mid;
        if (a[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1; // 没找到
}

关键点：mid = left + (right - left) / 2 而不是 (left + right) / 2， 是为了防止 left + right 溢出。这是竞赛中的标准写法。

4. 归并排序：分治的经典应用

归并排序的核心思想：把一个数组从中间分成两半，分别对两半排序，然后把两个有序数组合并成一个有序数组。

归并排序的伪代码 
void mergeSort(int a[], int left, int right) {
    if (left >= right) return;  // 只有一个元素，天然有序
    
    int mid = left + (right - left) / 2;
    mergeSort(a, left, mid);      // 排序左半部分
    mergeSort(a, mid + 1, right); // 排序右半部分
    merge(a, left, mid, right);   // 合并两个有序数组
}

L4阶段不要求完整手写出 merge() 函数，但要求学生理解：

  • 为什么可以"拆"？因为排序左半和右半不影响彼此
  • 为什么可以"合"？因为两个有序数组的合并是 O(n) 的
  • 总复杂度为什么是 O(n log n)？每一层拆分是 O(n)，总共有 log n 层

5. 二分思想的应用拓展

二分不只是查找，更是一种决策思维：

  • 二分答案：猜一个答案，验证是否可行，根据结果调整范围
  • 二分查找边界：找第一个 ≥ target 的位置，或最后一个 ≤ target 的位置
  • 快速幂：用二分思想把幂运算从 O(n) 降到 O(log n)
快速幂思想 
long long fastPow(long long a, long long n) {
    if (n == 0) return 1;
    long long half = fastPow(a, n / 2);
    if (n % 2 == 0) return half * half;
    else return half * half * a;
}

快速幂用分治把指数对半砍，这是后续数论模块的重要基础。

IOI教练常见教学陷阱

陷阱1：二分查找边界写错

left <= right 还是 left < right？ mid + 1 还是 mid？ 二分是细节杀手，必须让学生多写、多错、多总结。

陷阱2：一开始就要求手写完整归并排序

归并排序的 merge() 过程对L4学生来说较复杂。 建议先通过动画/图示理解思想，代码实现可以放到L4末或L5初作为复习挑战。

陷阱3：只讲二分查找不讲二分思想

二分查找只是二分思想的冰山一角。更重要的是让学生理解"通过对半砍来加速决策"的思维模式。

课后练习题

1

二分查找基础

入门

给定有序数组和 target，用二分查找判断 target 是否存在，返回下标或 -1。

2

查找左边界

重点

给定有序数组（可能有重复元素）和 target，返回 target 第一次出现的下标。如果不存在返回 -1。

3

猜数字游戏

重点

系统随机生成一个 1~1000 的数字，玩家用二分策略猜测，统计最多需要几次能猜中。

4

归并排序模拟

挑战

给定一个数组，手动画出归并排序的拆分和合并过程（不需要写代码）。

本课自测清单

  • 能说出分治的三个步骤：拆分、解决、合并

  • 理解为什么二分查找的时间复杂度是 O(log n)
  • 理解归并排序"先拆分后合并"的核心思想
  • 知道 mid = left + (right - left) / 2 的作用
  • 如果任何一条不满足，不要进入L5搜索期
