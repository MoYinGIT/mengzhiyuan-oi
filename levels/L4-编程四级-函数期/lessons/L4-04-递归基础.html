递归基础

递归三要素、递归执行过程、递归与迭代的对比

8课时
约2周
难度

本课三维目标

知识 递归定义、终止条件、递归调用、递归展开过程
技能 能独立写出阶乘、斐波那契、汉诺塔等经典递归程序
思维 建立"把问题交给更小的自己解决"的递归信念

递归是很多学生的"鬼门关"，也是竞赛生与业余爱好者的分水岭。

学不会递归的学生，到L6的搜索会寸步难行。但递归其实不需要"理解"，需要的是信任—— 信任"更小的子问题已经被解决"，你只需要负责把当前问题拆成更小的子问题。

我的教学策略是：先讲故事（汉诺塔、阶乘），再画展开图，最后写代码。 不要一上来就讲数学归纳法。

1. 什么是递归？

递归就是：函数调用自己。

听起来像绕口令？看一个最简单的例子：阶乘。

阶乘的递归写法 
long long factorial(int n) {
    if (n == 0 || n == 1) return 1;  // 终止条件
    return n * factorial(n - 1);      // 调用自己，但规模更小
}

factorial(5) 的计算过程：

graph TD A[factorial5] --> B[5 * factorial4] B --> C[4 * factorial3] C --> D[3 * factorial2] D --> E[2 * factorial1] E --> F[return 1] E --> G[return 2 * 1 = 2] D --> H[return 3 * 2 = 6] C --> I[return 4 * 6 = 24] B --> J[return 5 * 24 = 120]

2. 递归三要素

写一个正确的递归函数，必须包含三个要素：

  1. 终止条件（Base Case）
    什么时候停止递归？如果没有终止条件，函数会无限调用自己，导致栈溢出。
  2. 递归调用（Recursive Call）
    函数如何调用自己？每次调用的规模必须比当前更小，向终止条件靠近。
  3. 合并结果（Combine）
    如何利用子问题的结果，得到当前问题的结果？

教练口诀："终止、调用、合并"——缺一个，递归必错。

3. 递归的执行过程：调用栈

每次函数调用，系统都会在栈中分配一段空间，保存当前函数的参数和局部变量。 递归时，这些栈帧一层层叠加，直到触发终止条件，再一层层返回。

如果递归太深（比如 factorial(100000)），栈空间会被耗尽，程序崩溃。 这就是为什么有些问题虽然可以递归，但竞赛中更倾向于用迭代或记忆化。

4. 经典递归：汉诺塔

汉诺塔是讲解递归最好的故事之一。规则：三根柱子，n个盘子从上到下由小到大。 目标是把所有盘子从A柱移到C柱，每次只能移动一个盘子，大盘不能压小盘。

汉诺塔递归代码 
void hanoi(int n, char from, char temp, char to) {
    if (n == 1) {
        cout << from << " -> " << to << endl;
        return;
    }
    hanoi(n - 1, from, to, temp);    // 1. 把上面 n-1 个从 from 移到 temp
    cout << from << " -> " << to << endl; // 2. 把最底下的从 from 移到 to
    hanoi(n - 1, temp, from, to);    // 3. 把 n-1 个从 temp 移到 to
}

汉诺塔的核心思想：把"移动n个盘子"拆成"移动n-1个盘子"——这正是递归的精髓。

5. 递归 vs 迭代

任何可以用递归解决的问题，理论上也可以用迭代解决。两者的对比：

特性 递归 迭代
代码可读性 通常更简洁、贴近数学定义 可能需要更多辅助变量
空间复杂度 需要栈空间，可能溢出 通常只需要 O(1) 额外空间
适用场景 树、图、分治、搜索 线性遍历、动态规划

L4阶段不要求深入优化，重点是让学生"敢写递归"、"会写递归"。 递归转迭代的技巧在L5-L6再系统讲解。

IOI教练常见教学陷阱

陷阱1：试图"完全理解"递归的每一步

很多学生在脑子里模拟递归展开，越想越乱。 要告诉他们：写递归时只需要关注"当前层"和"终止条件"，不要试图追踪所有层。

陷阱2：忘记写终止条件

没有终止条件的递归会导致无限循环和栈溢出（Runtime Error）。 这是初学者写递归最常见的错误。

陷阱3：过早引入记忆化搜索

记忆化搜索是优化手段，不是入门内容。L4阶段先让学生写出裸递归， 体会到重复计算的慢，后面再教优化，印象更深。

课后练习题

1

递归求阶乘

入门

用递归实现 factorial(n)，并画出 n=4 时的调用栈。

2

递归求幂

入门

用递归实现 pow(a, n)，即求 a 的 n 次方（假设 n 是非负整数）。

3

汉诺塔

重点

输入 n，输出把 n 个盘子从 A 柱移到 C 柱的每一步移动方案。

4

递归翻转字符串

挑战

用递归实现字符串翻转。例如输入 "abc"，输出 "cba"。

  • 能写出汉诺塔的递归代码并解释其思想
  • 知道递归没有终止条件会导致栈溢出
