函数定义与调用

函数声明、返回值、调用过程、代码模块化

8课时
约2周
难度

本课三维目标

知识 函数定义语法、返回值类型、形参、函数调用流程
技能 能独立编写简单函数（如判断素数、求最大公约数），并在main中调用
思维 建立"一个函数只做一件事"的模块化思维

1. 为什么需要函数？

假设一个程序既要判断3个数是不是素数，又要判断另外5个数是不是素数：

不用函数：重复代码 
// 判断第一个数
bool isPrime1 = true;
for (int i = 2; i * i <= x1; i++) {
    if (x1 % i == 0) isPrime1 = false;
}

// 判断第二个数
bool isPrime2 = true;
for (int i = 2; i * i <= x2; i++) {
    if (x2 % i == 0) isPrime2 = false;
}

// ... 还要写第三、四、五、六、七、八遍？

用函数，同样的逻辑只写一次：

用函数：一次定义，多次调用 
bool isPrime(int x) {
    if (x < 2) return false;
    for (int i = 2; i * i <= x; i++) {
        if (x % i == 0) return false;
    }
    return true;
}

int main() {
    if (isPrime(a)) cout << "是素数" << endl;
    if (isPrime(b)) cout << "是素数" << endl;
    // ... 想判断多少次就调用多少次
}

函数的本质：把一段可复用的逻辑打包成一个"黑盒"，给它输入，它返回输出。

2. 函数的基本结构

函数定义四部分 
// 返回值类型  函数名(参数列表) {
//     函数体
//     return 返回值;
// }

int add(int a, int b) {
    int c = a + b;
    return c;
}

void sayHello() {
    cout << "Hello!" << endl;
    // void 表示没有返回值，不需要 return
}

记忆口诀："返名参体"——返回值类型、函数名、参数列表、函数体。

3. 函数调用过程

当程序调用一个函数时，会发生以下事情：

  1. 程序跳转到函数体开始执行
  2. 实参的值被复制给形参（值传递）
  3. 执行函数体中的代码
  4. 遇到 return 时，把结果带回调用处
  5. 程序回到调用位置继续执行

4. 函数的声明与定义

在C++中，函数必须先声明后使用。如果函数定义在 main 之后，需要在 main 之前先声明：

函数声明与定义分离 
// 声明（告诉编译器：这个函数存在）
int max(int a, int b);

int main() {
    cout << max(3, 5) << endl; // 可以正常调用
    return 0;
}

// 定义（真正的实现）
int max(int a, int b) {
    return a > b ? a : b;
}

声明也叫函数原型，只需要写返回值、函数名和参数类型，不需要写参数名（虽然写上更好读）。

5. 常用函数的模块化拆分

竞赛代码中，常见的模块化方式：

竞赛代码模块化示例 
// 工具函数
int max(int a, int b) { return a > b ? a : b; }
int min(int a, int b) { return a < b ? a : b; }
bool isPrime(int x) { /* ... */ }

// 核心逻辑函数
int solve() {
    // 读取数据、处理逻辑、返回结果
}

int main() {
    int t;
    cin >> t;
    while (t--) {
        cout << solve() << endl;
    }
    return 0;
}

把核心逻辑封装在 solve() 中，是竞赛代码的标准写法。这样 main 函数非常简洁，只负责输入输出调度。

IOI教练常见教学陷阱

陷阱1：所有代码塞进main函数

初学者习惯"从上到下一条线写到底"。要刻意训练他们： 每写一段逻辑，问问自己"这段逻辑能不能做成一个函数？"

陷阱2：函数名随意

不要用 f1()、f2() 这种无意义函数名。 好的函数名是代码的自文档化：isPrime()、getSum()、solve()。

陷阱3：一个函数做太多事

如果一个函数超过30行，就要警惕。尝试把它拆成"读数据"、"处理"、"输出"三个函数。
