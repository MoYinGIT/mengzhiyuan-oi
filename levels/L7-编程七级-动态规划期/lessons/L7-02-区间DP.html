区间DP

区间合并、石子合并、矩阵链乘、区间最值

10课时
约2.5周
难度

本课三维目标

知识 区间DP状态定义、枚举长度和分割点、边界条件
技能 能独立解决石子合并、矩阵链乘、括号匹配等经典区间DP问题
思维 建立"大区间由小区间合并而来"的递推思维

1. 什么是区间DP？

区间DP是一类以区间作为状态的动态规划。 它的核心思想是：一个区间 [l, r] 的答案，可以由它内部的更小区间通过某种规则合并得到。

典型的区间DP问题是石子合并：有一排石子，每次只能合并相邻的两堆， 代价是两堆石子的重量之和。求把所有石子合并成一堆的最小总代价。

2. 区间DP的标准套路

区间DP有两个固定的循环结构：

  1. 枚举区间长度 len（从小到大）
  2. 枚举左端点 l，计算右端点 r = l + len - 1
  3. 枚举分割点 k（l ≤ k < r），尝试把 [l, r] 拆成 [l, k] 和 [k+1, r]
区间DP标准框架 
// dp[l][r] = 区间[l,r]的最优值
for (int len = 2; len <= n; len++) {        // 枚举区间长度
    for (int l = 1; l + len - 1 <= n; l++) { // 枚举左端点
        int r = l + len - 1;                  // 计算右端点
        dp[l][r] = INF;                       // 初始化
        for (int k = l; k < r; k++) {         // 枚举分割点
            dp[l][r] = min(dp[l][r], 
                           dp[l][k] + dp[k+1][r] + mergeCost(l, r));
        }
    }
}

记忆口诀："长度从小到大，左端点从左到右，分割点扫一遍"。

3. 经典问题：石子合并

问题：n 堆石子排成一排，第 i 堆重量为 a[i]。 每次只能合并相邻的两堆，代价为合并后总重量。求合并成一堆的最小代价。

石子合并代码 
const int INF = 1e9;
int a[305], pre[305];
int dp[305][305];

// pre[i] 是前缀和，pre[i] = a[1] + ... + a[i]

for (int i = 1; i <= n; i++) {
    pre[i] = pre[i-1] + a[i];
    dp[i][i] = 0;  // 一堆石子不需要合并
}

for (int len = 2; len <= n; len++) {
    for (int l = 1; l + len - 1 <= n; l++) {
        int r = l + len - 1;
        dp[l][r] = INF;
        for (int k = l; k < r; k++) {
            int cost = dp[l][k] + dp[k+1][r] + pre[r] - pre[l-1];
            dp[l][r] = min(dp[l][r], cost);
        }
    }
}

cout << dp[1][n] << endl;

合并区间 [l, r] 的代价 = 左半部分最优代价 + 右半部分最优代价 + 区间总重量。 区间总重量用前缀和 O(1) 计算。

4. 矩阵链乘

问题：给定 n 个矩阵的维度，求把它们相乘所需的最少标量乘法次数。 矩阵乘法满足结合律，不同的结合顺序计算量可能差别巨大。

矩阵链乘代码 
// p[i] 是第i个矩阵的行数，p[i+1]是列数
// 矩阵i的维度是 p[i] × p[i+1]

for (int len = 2; len <= n; len++) {
    for (int l = 1; l + len - 1 <= n; l++) {
        int r = l + len - 1;
        dp[l][r] = INF;
        for (int k = l; k < r; k++) {
            // 在k处分割：左边[l,k]，右边[k+1,r]
            int cost = dp[l][k] + dp[k+1][r] + p[l] * p[k+1] * p[r+1];
            dp[l][r] = min(dp[l][r], cost);
        }
    }
}

和石子合并的结构几乎完全一样，只是合并代价的计算公式不同。 这正是区间DP的模板化特征。

5. 回文串与括号匹配

区间DP还可以用来解决字符串问题。例如：求一个字符串中最长回文子序列的长度。

最长回文子序列 
// dp[l][r] = s[l..r]中最长回文子序列的长度
for (int i = 1; i <= n; i++) dp[i][i] = 1;

for (int len = 2; len <= n; len++) {
    for (int l = 1; l + len - 1 <= n; l++) {
        int r = l + len - 1;
        if (s[l] == s[r]) {
            dp[l][r] = dp[l+1][r-1] + 2;
        } else {
            dp[l][r] = max(dp[l+1][r], dp[l][r-1]);
        }
    }
}

如果两端字符相同，可以同时选；否则，舍弃左端或右端中的一个，取更优者。

IOI教练常见教学陷阱

陷阱1：区间长度从1开始

很多区间DP中长度为1的答案是已知的（如dp[i][i]=0）。 循环应从len=2开始，否则分割点k的循环会出问题。

陷阱2：右端点计算错误

右端点 r = l + len - 1，不是 l + len。 这个off-by-one错误很常见。

陷阱3：只记模板不理解合并代价

不同问题的合并代价公式不同。要让学生自己推导： 把 [l,k] 和 [k+1,r] 合并起来，代价是什么？

重点

括号匹配

挑战

给定括号序列，求最少添加多少个括号使其完全匹配。（用区间DP）
