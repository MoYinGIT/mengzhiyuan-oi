背包问题

0/1背包、完全背包、多重背包、空间优化、经典变形

10课时
约2.5周
难度

本课三维目标

知识 0/1背包、完全背包、多重背包的状态定义与转移方程
技能 能独立写出三种背包的代码，掌握一维数组空间优化
思维 建立"容量作为状态维度"的背包建模思维

1. 0/1背包问题

问题：有 n 个物品和一个容量为 V 的背包。每个物品只能选一次， 第 i 个物品的体积是 w[i]，价值是 v[i]。 求在不超过背包容量的前提下，能装下的最大价值。

状态定义：dp[i][j] 表示前 i 个物品，在容量为 j 的背包中能获得的最大价值。

状态转移：对于第 i 个物品，有两种选择：

  • 不选：dp[i][j] = dp[i-1][j]
  • 选：dp[i][j] = dp[i-1][j-w[i]] + v[i]（前提是 j ≥ w[i]）
0/1背包二维代码 
// 初始化：前0个物品，价值都是0
for (int j = 0; j <= V; j++) dp[0][j] = 0;

for (int i = 1; i <= n; i++) {
    for (int j = 0; j <= V; j++) {
        dp[i][j] = dp[i-1][j];  // 不选第i个
        if (j >= w[i]) {
            dp[i][j] = max(dp[i][j], dp[i-1][j-w[i]] + v[i]);
        }
    }
}

2. 空间优化：一维数组

观察发现，dp[i][j] 只依赖于 dp[i-1][...]， 所以可以把二维优化成一维数组。

关键技巧：容量 j 必须倒序遍历！ 如果正序遍历，同一轮中前面的更新会影响到后面，相当于一个物品被选了多次。

0/1背包一维优化 
int dp[MAXV] = {0};

for (int i = 1; i <= n; i++) {
    for (int j = V; j >= w[i]; j--) {  // 必须倒序！
        dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
    }
}

3. 完全背包

完全背包中，每个物品可以选无限次。状态转移方程和0/1背包很像， 但容量 j 需要正序遍历！

完全背包一维优化 
int dp[MAXV] = {0};

for (int i = 1; i <= n; i++) {
    for (int j = w[i]; j <= V; j++) {  // 正序！允许重复选
        dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
    }
}

记忆口诀：0/1背包倒着走（每个只选一次），完全背包正着走（可以重复选）。

4. 多重背包

多重背包中，每个物品最多可以选 c[i] 次。 最简单的做法是把它拆成 c[i] 个0/1背包物品，但这样效率太低。

二进制优化：把 c[i] 拆成 1, 2, 4, ..., 剩余部分。 这样任何数量都可以由这些数组合而成，且拆分后的物品数从 O(c) 降到 O(log c)。

多重背包二进制优化 
// 把每个物品拆分成若干"新物品"
vector<pair<int, int>> items;  // (weight, value)

for (int i = 1; i <= n; i++) {
    int cnt = c[i];
    int k = 1;
    while (cnt > 0) {
        int use = min(k, cnt);
        items.push_back({w[i] * use, v[i] * use});
        cnt -= use;
        k *= 2;
    }
}

// 然后对这些"新物品"做0/1背包
for (auto [weight, value] : items) {
    for (int j = V; j >= weight; j--) {
        dp[j] = max(dp[j], dp[j - weight] + value);
    }
}

5. 背包问题的常见变形

  • 恰好装满：初始化时，dp[0] = 0，其余为 -INF。 这样只有恰好装满的容量才会被更新。
  • 求方案数：把 max 改成求和，记录达到最大价值的方案数量。
  • 求具体方案：用二维数组记录每个状态是从哪个决策转移来的，最后回溯输出。
  • 分组背包：物品分成若干组，每组最多选一个。对每组做一次0/1背包决策。

IOI教练常见教学陷阱

陷阱1：0/1背包正序遍历

这是背包问题最高频的错误。正序遍历会让同一个物品被多次使用， 把0/1背包变成了完全背包。必须反复强调倒序的原因。

陷阱2：容量初始化错误

"不超过容量求最大价值"时全部初始化为0；"恰好装满"时dp[0]=0，其余为-INF。 两种场景初始化不同，写错会导致答案错误。

陷阱3：只会背模板不会识别模型

很多题目不会直接说"背包"。比如"在总时间限制内获得最大分数"、"在预算内买最多东西"—— 这些都是背包。要训练学生的建模能力。
