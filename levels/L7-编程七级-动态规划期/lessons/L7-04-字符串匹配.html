字符串匹配

KMP算法、next数组、Trie树入门、字符串哈希

8课时
约2周
难度

本课三维目标

知识 KMP算法原理、next数组定义、Trie树结构、字符串哈希
技能 能独立写出KMP的next数组和匹配代码，会用字符串哈希判断子串相等
思维 理解"利用已匹配信息避免回退"的KMP核心思想

1. 朴素匹配的缺陷

给定文本串 S 和模式串 P，判断 P 是否在 S 中出现。 朴素做法是：把 P 依次对准 S 的每个位置，逐个字符比较。 最坏时间复杂度 O(|S| × |P|)。

当匹配失败时，朴素算法会把 P 向右移动一位，然后从头开始比较。 但很多时候，我们已经知道前面的一部分字符是匹配成功的， 这些已知信息应该被利用起来，而不是浪费掉。

上图所示的情况，朴素算法会把 P 右移1位，但实际上我们可以直接右移2位， 因为前几位 "ababa" 的自身结构告诉我们：移1位不可能成功。

2. next数组的本质

next[i] 表示：模式串 P[0..i-1] 中，最长的相等前后缀的长度。

例如 P = "ababa"：

  • 前缀："a", "ab", "aba", "abab"
  • 后缀："a", "ba", "aba", "baba"
  • 最长的相等前后缀是 "aba"，长度为3

当 P[5] 匹配失败时，我们知道前5个字符 "ababa" 已经匹配了。 由于 "ababa" 的最长相等前后缀是 "aba"（长度3）， 所以 P 可以直接跳到 next[5] = 3 的位置继续比较， 而不需要回退文本串的指针。

3. 求next数组

KMP求next数组 
vector<int> getNext(string& p) {
    int m = p.size();
    vector<int> next(m, 0);
    
    for (int i = 1, j = 0; i < m; i++) {
        while (j > 0 && p[i] != p[j]) {
            j = next[j - 1];  // 回退到前一个匹配位置
        }
        if (p[i] == p[j]) {
            j++;
        }
        next[i] = j;
    }
    
    return next;
}

next数组的求解本身也是一个"匹配"过程：用 P 的前缀去匹配 P 的后缀。 这是KMP算法最精妙的部分。

4. KMP匹配过程

KMP匹配 
vector<int> kmpSearch(string& s, string& p) {
    vector<int> next = getNext(p);
    vector<int> result;
    int n = s.size(), m = p.size();
    
    for (int i = 0, j = 0; i < n; i++) {
        while (j > 0 && s[i] != p[j]) {
            j = next[j - 1];
        }
        if (s[i] == p[j]) {
            j++;
        }
        if (j == m) {
            result.push_back(i - m + 1);  // 匹配成功
            j = next[j - 1];  // 继续寻找下一个匹配
        }
    }
    
    return result;
}

KMP的时间复杂度是 O(|S| + |P|)，这是字符串匹配算法中最重要的基础算法之一。

5. 字符串哈希（拓展）

字符串哈希是把一个字符串映射成一个整数，从而可以在 O(1) 时间内比较两个子串是否相等。

字符串哈希模板 
const long long BASE = 131;
const long long MOD = 1e9 + 7;

long long h[MAXN], p[MAXN];

void initHash(string& s) {
    int n = s.size();
    p[0] = 1;
    for (int i = 1; i <= n; i++) {
        h[i] = (h[i-1] * BASE + s[i-1]) % MOD;
        p[i] = (p[i-1] * BASE) % MOD;
    }
}

// 求子串 s[l..r] 的哈希值 (1-based, 闭区间)
long long getHash(int l, int r) {
    return ((h[r] - h[l-1] * p[r-l+1]) % MOD + MOD) % MOD;
}

字符串哈希在判断回文串、最长重复子串、子串去重等问题中非常高效。

6. Trie树入门（拓展）

Trie（前缀树）是一种树形数据结构，用于高效地存储和查找字符串集合。 它可以快速判断一个字符串是否是某个给定字符串的前缀。

Trie树基础结构 
const int MAXN = 100005;
int trie[MAXN][26];  // 假设只有小写字母
bool isEnd[MAXN];
int nodeCount = 0;

void insert(string& s) {
    int p = 0;
    for (char c : s) {
        int idx = c - 'a';
        if (trie[p][idx] == 0) {
            trie[p][idx] = ++nodeCount;
        }
        p = trie[p][idx];
    }
    isEnd[p] = true;
}

Trie树在自动补全、拼写检查、多模式匹配（AC自动机的基础）中有广泛应用。

IOI教练常见教学陷阱

陷阱1：next数组下标从0还是从1开始搞混

不同教材对next的定义不同。有些从1开始，有些从0开始。 我们的模板采用0-based字符串，next[i]表示P[0..i]的最长相等前后缀长度。 要让学生统一用一种理解方式。

陷阱2：死记硬背next数组代码

next数组的推导是KMP的灵魂。如果学生不理解"为什么j=next[j-1]"， 遇到next数组的变体题（如求周期、求border）就会懵。

陷阱3：KMP和哈希混用场景不清

KMP适合单模式串匹配，哈希适合多子串比较。 要让学生根据题目特点选择：求出现次数、位置用KMP；比较子串相等用哈希。
