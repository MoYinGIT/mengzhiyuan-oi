图的DFS与BFS

连通分量、判环、拓扑序检测、图的遍历模板

8课时
约2周
难度

本课三维目标

知识 图的DFS/BFS模板、连通分量、判环原理、入度出度
技能 能用DFS/BFS求连通分量数量、判断图是否存在环
思维 理解图上遍历与网格遍历的本质联系与差异

1. 图上的DFS模板

图上的DFS与网格上的DFS思想完全一致，只是"邻居"的定义从上下左右变成了邻接表中的节点。

图的DFS模板 
const int MAXN = 100005;
vector<int> adj[MAXN];
bool visited[MAXN];

void dfs(int u) {
    visited[u] = true;
    for (int v : adj[u]) {
        if (!visited[v]) {
            dfs(v);
        }
    }
}

核心依然是：标记访问 → 遍历邻居 → 递归深入。不同的是，图的邻居由输入决定，而不是固定的四个方向。

2. 连通分量计数

在无向图中，如果任意两点之间都有路径相连，这个子图就是一个连通分量。 求连通分量数量的方法和L5的岛屿计数如出一辙。

连通分量计数 
int countComponents(int n) {
    int count = 0;
    for (int i = 1; i <= n; i++) {
        if (!visited[i]) {
            dfs(i);
            count++;
        }
    }
    return count;
}

每发现一个未被访问的节点，就从这个节点开始一次DFS，所有能到达的节点都属于同一个连通分量。

3. 图的判环

无向图判环：在DFS中，如果从节点u走到节点v，而v已经被访问过且v不是u的父亲节点， 说明存在环。

无向图判环 
bool hasCycle = false;

void dfs(int u, int parent) {
    visited[u] = true;
    for (int v : adj[u]) {
        if (!visited[v]) {
            dfs(v, u);
        } else if (v != parent) {
            // v已被访问且不是父节点，说明有环
            hasCycle = true;
        }
    }
}

有向图判环：需要使用三种状态标记：0=未访问，1=正在访问（在当前递归栈中），2=已处理完。 如果在DFS中遇到一个状态为1的节点，说明存在回边，即存在环。

有向图判环 
int state[MAXN];  // 0=未访问, 1=访问中, 2=已结束
bool hasCycle = false;

void dfs(int u) {
    state[u] = 1;  // 标记为访问中
    for (int v : adj[u]) {
        if (state[v] == 0) {
            dfs(v);
        } else if (state[v] == 1) {
            // 遇到回边，存在环
            hasCycle = true;
        }
    }
    state[u] = 2;  // 标记为已结束
}

4. 图上的BFS

图上的BFS同样使用队列，与L5的网格BFS完全一致。BFS在图上常用于求单源最短路径（无权图）。

图的BFS模板 
int dist[MAXN];

void bfs(int start) {
    queue<int> q;
    for (int i = 1; i <= n; i++) dist[i] = -1;
    
    dist[start] = 0;
    q.push(start);
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        
        for (int v : adj[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}

BFS结束后，dist[v] 就是从起点到v的最短路径长度（边数）。 这是无权图上求最短路径的标准做法。

5. 二分图判定（拓展）

二分图是指顶点可以被分成两组，使得每条边的两个端点都在不同组中。 可以用DFS/BFS进行二色染色判定。

二分图判定 
int color[MAXN];  // 0=未染色, 1=颜色A, -1=颜色B
bool isBipartite = true;

void dfs(int u, int c) {
    color[u] = c;
    for (int v : adj[u]) {
        if (color[v] == 0) {
            dfs(v, -c);
        } else if (color[v] == c) {
            isBipartite = false;
        }
    }
}

IOI教练常见教学陷阱

陷阱1：无向图判环时忽略父节点

在无向图中，u-v是双向边。如果不记录父节点，DFS会认为自己从u到了v， 然后v又"发现"u已被访问，误判为环。

陷阱2：有向图判环只用visited数组

有向图的环检测必须用三色标记法（0/1/2）。单纯的visited数组无法区分"祖先节点"和"已经处理完的分支"。

陷阱3：图不连通时只从一个点开始遍历

很多图可能由多个连通分量组成。遍历图时要用外层循环扫描所有节点，确保不遗漏。

连通分量数量
