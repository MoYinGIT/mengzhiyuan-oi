最小生成树

Kruskal、Prim、并查集、贪心正确性理解

8课时
约2周
难度

本课三维目标

知识 最小生成树定义、Kruskal算法、Prim算法、并查集
技能 能独立写出Kruskal算法（含并查集），理解Prim的核心思想
思维 理解"贪心选择+安全边定理"在MST问题中的正确性保证

1. 什么是最小生成树？

给定一个连通的无向带权图，最小生成树（MST）是包含所有顶点的一棵子树， 使得树上所有边的权重之和最小。

生活中的例子：要在n个城市之间铺设电缆，每条电缆有不同的造价。 如何选择电缆使得所有城市连通，且总造价最低？这就是最小生成树问题。

2. Kruskal算法

Kruskal算法的核心思想是贪心：每次从所有边中选择权重最小的一条， 如果这条边的两个端点不在同一个连通块中，就把这条边加入生成树。

Kruskal模板 
#include <algorithm>
using namespace std;

struct Edge {
    int u, v, w;
    bool operator<(const Edge& other) const {
        return w < other.w;
    }
};

vector<Edge> edges;
int parent[MAXN];

// 并查集
int find(int x) {
    return parent[x] == x ? x : parent[x] = find(parent[x]);
}

void unite(int x, int y) {
    parent[find(x)] = find(y);
}

long long kruskal(int n) {
    sort(edges.begin(), edges.end());  // 按权重排序
    
    for (int i = 1; i <= n; i++) parent[i] = i;
    
    long long total = 0;
    int count = 0;
    
    for (auto& e : edges) {
        if (find(e.u) != find(e.v)) {
            unite(e.u, e.v);
            total += e.w;
            count++;
            if (count == n - 1) break;  // 已经选了n-1条边
        }
    }
    
    return total;
}

为什么正确？这是基于安全边定理（Cut Property）： 对于图的任意一个割，权重最小的横切边一定属于某个最小生成树。 Kruskal每次都选全局最小边，这条边必然是某个割的安全边。

3. 并查集：Kruskal的好搭档

并查集（Disjoint Set Union, DSU）用来高效判断两个顶点是否在同一个连通块中， 以及合并两个连通块。

并查集通过路径压缩优化，使得 find 操作接近 O(1)。 这是竞赛中处理连通性问题的标准数据结构。

4. Prim算法

Prim算法的思路是从一个顶点开始，逐步扩展生成树。 每次选择与当前树相连的最小权重边，把对应的顶点加入树中。

Prim模板（堆优化） 
long long prim(int n) {
    vector<bool> inMST(n + 1, false);
    vector<long long> minDist(n + 1, INF);
    
    minDist[1] = 0;
    priority_queue<pair<long long, int>, 
                    vector<pair<long long, int>>, 
                    greater<>> pq;
    pq.push({0, 1});
    
    long long total = 0;
    int count = 0;
    
    while (!pq.empty() && count < n) {
        auto [d, u] = pq.top();
        pq.pop();
        
        if (inMST[u]) continue;
        
        inMST[u] = true;
        total += d;
        count++;
        
        for (auto [v, w] : adj[u]) {
            if (!inMST[v] && w < minDist[v]) {
                minDist[v] = w;
                pq.push({w, v});
            }
        }
    }
    
    return (count == n) ? total : -1;  // -1表示图不连通
}

Prim适合稠密图（边很多），因为它不需要存储边集，直接用邻接表扩展。 但在稀疏图上，Kruskal通常更简洁高效。

5. Kruskal vs Prim

特性 Kruskal Prim
核心思想 按边贪心，合并连通块 按顶点扩展，加入最近顶点
数据结构 并查集 + 排序 优先队列 + 邻接表
适用图 稀疏图 稠密图
竞赛偏好 更常用，代码短 特定场景使用

IOI教练常见教学陷阱

陷阱1：并查集没有路径压缩

没有路径压缩的并查集find操作是O(log n)甚至O(n)，在边数很多时会TLE。 一定要教学生写 parent[x] = find(parent[x])。

陷阱2：图不连通时没有特判

如果图不连通，不存在最小生成树。Kruskal结束后要检查count是否等于n-1。

陷阱3：只记模板不理解贪心正确性

Kruskal的贪心不是随便猜的，它有严格的数学保证（安全边定理）。 让学生知道这一点，有助于他们理解为什么这个算法永远正确。

课后练习题

1

最小生成树（Kruskal）

入门

给定无向带权图，求最小生成树的权重和。如果图不连通，输出-1。

2

并查集基础

入门

实现并查集，支持合并两个集合和查询两个元素是否属于同一集合。

3

次小生成树（拓展）

挑战

求严格次小生成树的权重（即所有生成树中，权重仅次于MST的那个）。

4

连接所有点的最小费用

重点

平面上有n个点，两点之间的边权为曼哈顿距离。求连接所有点的最小总费用。
