拓扑排序

Kahn算法、DFS拓扑序、DAG上的DP

8课时
约2周
难度

本课三维目标

知识 拓扑排序定义、Kahn算法、DFS求拓扑序、DAG概念
技能 能用Kahn算法或DFS求拓扑序，判断有向图是否存在环
思维 理解拓扑序与依赖关系、任务调度、前置条件的天然联系

1. 什么是拓扑排序？

拓扑排序是对有向无环图（DAG）的顶点进行线性排序，使得对于图中的每一条有向边 u→v， u 在排序中都出现在 v 的前面。

换句话说，拓扑排序就是把有依赖关系的事情排一个先后顺序：先做A，再做B，最后做C。 生活中的例子：课程选修计划（先学C语言再学数据结构）、软件编译顺序（先编译依赖库再编译主程序）、项目管理（先完成前置任务）。

注意：如果图中存在环，就不存在拓扑排序。因为环上的节点互相依赖，无法确定先后顺序。

2. Kahn算法（BFS版拓扑排序）

Kahn算法的核心思想： repeatedly找到入度为0的顶点，把它加入拓扑序列，然后删除它以及从它出发的所有边。

Kahn算法模板 
vector<int> topoSort(int n) {
    vector<int> inDegree(n + 1, 0);
    
    // 计算每个点的入度
    for (int u = 1; u <= n; u++) {
        for (int v : adj[u]) {
            inDegree[v]++;
        }
    }
    
    queue<int> q;
    for (int i = 1; i <= n; i++) {
        if (inDegree[i] == 0) q.push(i);
    }
    
    vector<int> result;
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        result.push_back(u);
        
        for (int v : adj[u]) {
            inDegree[v]--;
            if (inDegree[v] == 0) {
                q.push(v);
            }
        }
    }
    
    // 如果结果中节点数 < n，说明有环
    if (result.size() < n) {
        // 存在环，无拓扑序
    }
    
    return result;
}

如果最后 result.size() == n，说明成功得到了拓扑序； 否则图中存在环。

3. DFS求拓扑序

除了Kahn算法，还可以用DFS求拓扑序。思路是： 对一个节点u，先递归处理它所有的后继节点，然后把u加入序列。 最后把序列反转，就是拓扑序。

DFS拓扑排序 
vector<int> topo;
int state[MAXN];  // 0=未访问, 1=访问中, 2=已完成
bool hasCycle = false;

void dfs(int u) {
    state[u] = 1;
    for (int v : adj[u]) {
        if (state[v] == 0) {
            dfs(v);
        } else if (state[v] == 1) {
            hasCycle = true;  // 遇到回边，有环
        }
    }
    state[u] = 2;
    topo.push_back(u);  // 后序遍历加入
}

// 调用完成后，reverse(topo.begin(), topo.end()) 即为拓扑序

DFS版拓扑排序天然支持判环，和L6-02中有向图判环的方法完全一致。

4. DAG上的动态规划

拓扑排序最重要的应用之一，就是在DAG上进行动态规划。 因为拓扑序保证了处理某个节点时，它的所有前驱节点都已经被处理过了。

DAG最长路径 
// 先求拓扑序
vector<int> order = topoSort(n);

// dp[u] 表示从起点到u的最长路径长度
vector<long long> dp(n + 1, -INF);
dp[start] = 0;

for (int u : order) {
    if (dp[u] == -INF) continue;  // 不可达
    for (int v : adj[u]) {
        if (dp[u] + w(u,v) > dp[v]) {
            dp[v] = dp[u] + w(u,v);
        }
    }
}

DAG上的DP不需要像一般图那样用SPFA或Bellman-Ford， 只需要按拓扑序递推，复杂度是 O(n+m)。 这是图论与DP结合的最经典场景之一。

5. Kahn vs DFS拓扑排序

特性 Kahn算法 DFS拓扑序
数据结构 队列 + 入度数组 递归栈 + 三色标记
代码长度 稍长 稍短
竞赛偏好 更直观，适合初学者 与DFS判环天然结合

IOI教练常见教学陷阱

陷阱1：有向图存在环仍求拓扑序

有环图不存在拓扑序。很多学生在Kahn算法结束后不检查结果大小， 直接输出一个不完整的序列，导致WA。

陷阱2：入度更新遗漏

Kahn算法中，删除节点u后，必须把所有u的出边邻居的入度减1。 遗漏一条边就会导致后续判断错误。

陷阱3：DFS拓扑序忘记reverse

DFS得到的是后序遍历序列，必须反转才是拓扑序。 很多学生漏掉reverse这一步。

重点
