图的表示

邻接矩阵、邻接表、边集、图的存储方式选择

8课时
约2周
难度

本课三维目标

知识 邻接矩阵、邻接表（vector/数组）、边集结构、有向图与无向图
技能 能根据顶点数和边数选择合适的存储结构，完成图的读入
思维 理解"存储结构决定算法效率"的权衡思维

1. 什么是图？

图（Graph）由顶点（Vertex）和边（Edge）组成，用来描述事物之间的关系。 比如城市是顶点，道路是边；人是顶点，社交关系是边。

图可以分为：

  • 有向图：边有方向，A→B 不代表 B→A
  • 无向图：边没有方向，A-B 意味着两边互通
  • 带权图：边上有数值（如距离、费用）

2. 邻接矩阵

用一个 n×n 的矩阵 g[i][j] 表示顶点 i 到顶点 j 是否有边（或边的权重）。

邻接矩阵示例 
const int INF = 1e9;
int g[505][505];  // 邻接矩阵

// 初始化
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= n; j++)
        g[i][j] = (i == j) ? 0 : INF;

// 读入边
for (int i = 0; i < m; i++) {
    int u, v, w;
    cin >> u >> v >> w;
    g[u][v] = min(g[u][v], w);  // 处理重边
    if (!directed) g[v][u] = min(g[v][u], w);
}

优点：O(1)查询任意两点间是否有边
缺点：空间复杂度 O(n²)，不适合稀疏图（边很少的图）

3. 邻接表

对每个顶点，用一个列表存储所有与它直接相连的邻居。竞赛中最常用的是 vector 实现邻接表。

vector邻接表 
#include <vector>
using namespace std;

const int MAXN = 100005;
vector<pair<int, int>> adj[MAXN];  // adj[u] = {(v, w), ...}

// 读入边
for (int i = 0; i < m; i++) {
    int u, v, w;
    cin >> u >> v >> w;
    adj[u].push_back({v, w});
    if (!directed) adj[v].push_back({u, w});
}

// 遍历u的所有邻居
for (auto [v, w] : adj[u]) {
    cout << u << " -> " << v << " 权重: " << w << endl;
}

优点：空间复杂度 O(n+m)，适合稀疏图，遍历邻居效率高
缺点：查询特定边是否存在需要 O(degree)

4. 边集表示

有些算法（如Kruskal）只需要边的信息，不需要快速查询邻接关系。这时可以用简单的边集存储。

边集结构 
struct Edge {
    int u, v, w;
};

vector<Edge> edges;

// 读入
for (int i = 0; i < m; i++) {
    int u, v, w;
    cin >> u >> v >> w;
    edges.push_back({u, v, w});
}

5. 如何选择存储方式？

场景 推荐存储 原因
n ≤ 500，稠密图 邻接矩阵 查询快，空间够用
n ≤ 10^5，稀疏图 邻接表 空间省，遍历快
最小生成树（Kruskal） 边集 只需要按权重排序边

IOI教练常见教学陷阱

陷阱1：有向图和无向图存混

无向图存邻接表时必须双向添加边。很多学生只加了一条，导致遍历结果只有一半。

陷阱2：重边不处理

输入中两点之间可能有多条边。最短路问题中要保留权重最小的那条边。 邻接矩阵用min更新，邻接表可以都存（后续算法会自然处理）或只保留最小边。

陷阱3：自环和重边搞混

自环（u→u）在大多数最短路径算法中没有意义，但某些题目会故意给。要教学生识别并忽略不必要的自环。

课后练习题

1

图的读入与输出

入门

读入一个有向带权图，分别用邻接矩阵和邻接表存储，并输出每个点的出边。

2

判断完全图

入门

给定无向图，判断它是否为完全图（任意两点之间都有边）。

3

求每个点的度数

重点

给定无向图，输出每个顶点的度数（即与之相连的边数）。分别用邻接矩阵和邻接表实现并对比效率。

4

存储方式转换 重点

编写函数，将邻接矩阵转换为邻接表，并分析时间复杂度。
