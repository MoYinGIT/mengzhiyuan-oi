最短路径

Dijkstra、Bellman-Ford、SPFA、Floyd、算法选择

8课时
约2周
难度

本课三维目标

知识 Dijkstra、SPFA、Floyd算法原理、适用条件、时间复杂度
技能 能根据图的特点（有无负权、单源/多源）选择并写出正确的最短路径算法
思维 建立"贪心-松弛-动态规划"三种不同策略解决同一问题的对比思维

1. 最短路径问题分类

最短路径是图论中最经典的问题之一。不同的问题场景需要不同的算法：

算法 适用图 源点 时间复杂度
Dijkstra 非负权图 单源 O((n+m)log n)
SPFA 可有负权图 单源 O(km)，最坏O(nm)
Bellman-Ford 可有负权图 单源 O(nm)
Floyd 任意图 多源 O(n³)

2. Dijkstra算法

Dijkstra算法适用于边权非负的图。它的核心思想是贪心： 每次从"未确定最短距离的节点"中选择距离最小的那个，然后用它去更新邻居的距离。

Dijkstra模板（堆优化） 
#include <queue>
#include <vector>
using namespace std;

const long long INF = 1e18;
vector<pair<int, int>> adj[MAXN];
long long dist[MAXN];

void dijkstra(int start) {
    for (int i = 1; i <= n; i++) dist[i] = INF;
    dist[start] = 0;
    
    // 优先队列：按距离从小到大（使用greater）
    priority_queue<pair<long long, int>, 
                    vector<pair<long long, int>>, 
                    greater<pair<long long, int>>> pq;
    pq.push({0, start});
    
    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();
        
        if (d > dist[u]) continue;  // 过期数据，跳过
        
        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
}

注意：Dijkstra不能处理负权边！ 因为贪心策略依赖"当前最小距离已经确定"，负权边会破坏这个假设。

3. SPFA算法

SPFA是Bellman-Ford的队列优化版本，可以处理负权边，还能判负环。 在竞赛中，它是单源最短路径最常用的"万金油"算法。

SPFA模板 
const long long INF = 1e18;
long long dist[MAXN];
int cnt[MAXN];  // 记录每个点入队次数，用于判负环
bool inQueue[MAXN];

bool spfa(int start) {
    for (int i = 1; i <= n; i++) {
        dist[i] = INF;
        inQueue[i] = false;
        cnt[i] = 0;
    }
    
    dist[start] = 0;
    queue<int> q;
    q.push(start);
    inQueue[start] = true;
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        inQueue[u] = false;
        
        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                if (!inQueue[v]) {
                    q.push(v);
                    inQueue[v] = true;
                    cnt[v]++;
                    if (cnt[v] >= n) return false; // 存在负环
                }
            }
        }
    }
    return true;  // 无负环
}

SPFA的平均性能很好，但在特殊构造的数据上可能退化成 O(nm)。 正式比赛中，如果图没有负权边，优先用Dijkstra；如果有负权边，再用SPFA。

4. Floyd算法

Floyd算法基于动态规划，能一次性求出所有点对之间的最短路径。 它适用于顶点数较少的图（通常 n ≤ 500）。

Floyd模板 
const int INF = 1e9;
int dist[505][505];

void floyd(int n) {
    for (int k = 1; k <= n; k++)
        for (int i = 1; i <= n; i++)
            for (int j = 1; j <= n; j++)
                if (dist[i][k] + dist[k][j] < dist[i][j])
                    dist[i][j] = dist[i][k] + dist[k][j];
}

核心思想：逐步允许经过更多的中间点来优化路径。 dist[i][j] 表示只经过编号 ≤ k 的中间点时，i到j的最短距离。

IOI教练常见教学陷阱

陷阱1：Dijkstra用于负权图

这是最常见的错误。Dijkstra在负权图上会输出错误结果。 教学中要反复强调：看到负权边，立刻排除Dijkstra。

陷阱2：Floyd用于大图

Floyd是O(n³)，n=1000时就要执行10^9次操作，肯定会TLE。 要让学生建立"n>500不用Floyd"的条件反射。

陷阱3：只记模板不理解原理

很多学生能背下Dijkstra的代码，但不知道为什么优先队列要跳过过期数据。 不理解原理，遇到变体题（如次短路、第K短路）就会束手无策。

课后练习题

1

单源最短路（Dijkstra）

入门

给定非负权有向图和起点s，求s到所有其他点的最短距离。

2

带负权边的最短路

重点

给定可能有负权边的有向图，用SPFA求单源最短路径，并判断是否存在负环。

3

多源最短路

重点

给定小规模图（n ≤ 300），求任意两点间的最短路径长度。

4

最短路径计数

挑战

在求最短路径的同时，统计从起点到终点的最短路径有多少条。注意对结果取模。
