前缀和

前缀和定义、预处理、区间查询、空间换时间

10课时
约2.5周
难度

本课三维目标

知识 前缀和定义、递推公式、区间求和公式
技能 能独立写出前缀和预处理代码，并用O(1)回答区间查询
思维 建立"预处理换查询效率"的空间换时间思维

前缀和是我最推荐在L3教的内容之一，它是数组DP线的正式起点。

很多机构把前缀和放在L5-L6才讲，但我坚持L3就引入。 因为前缀和与一维数组天然契合，学生一旦掌握，就能用O(1)解决区间查询问题—— 这是他们第一次体验到"空间换时间"的算法魅力。

1. 问题引入：区间求和

假设有一个数组，存储了某班级30天的数学平均分：

原始数组 
int score[6] = {5, 3, 8, 2, 7, 4};
// 下标:           0  1  2  3  4  5

现在老师频繁提问："第2天到第4天的总分是多少？"

朴素做法：每次用循环加一遍，复杂度 O(n)。

但如果提问10000次呢？总复杂度变成 O(n × q)，太慢了。

前缀和就是解决这类问题的利器：先花 O(n) 预处理，之后每次查询 O(1)。

2. 前缀和的定义

前缀和数组 pre[i] 表示原数组从第0个元素到第 i 个元素的累加和（包含两端）。

前缀和公式 
pre[0] = a[0];
pre[1] = a[0] + a[1];
pre[2] = a[0] + a[1] + a[2];
// ...
pre[i] = pre[i-1] + a[i];  // 递推公式！

3. 区间求和的 O(1) 查询

要求 a[l] + a[l+1] + ... + a[r] 的和，用前缀和只需要一步：

区间求和公式 
sum(l, r) = pre[r] - pre[l-1];  // 当 l > 0 时
sum(0, r) = pre[r];             // 当 l == 0 时

例如求 a[2] + a[3] + a[4]：

  • pre[4] = 5+3+8+2+7 = 25
  • pre[1] = 5+3 = 8
  • sum = 25 - 8 = 17（即 8+2+7 = 17）✅

记忆口诀：前缀和做差，右端点减去左端点前一个。 pre[r] - pre[l-1]。

4. 完整代码模板

前缀和模板代码 
const int MAXN = 100005;
int a[MAXN];
long long pre[MAXN]; // 注意：前缀和可能爆int

int main() {
    int n;
    cin >> n;
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }
    
    // 预处理前缀和
    pre[0] = a[0];
    for (int i = 1; i < n; i++) {
        pre[i] = pre[i-1] + a[i];
    }
    
    // 回答q次查询
    int q;
    cin >> q;
    while (q--) {
        int l, r;
        cin >> l >> r;
        if (l == 0) {
            cout << pre[r] << endl;
        } else {
            cout << pre[r] - pre[l-1] << endl;
        }
    }
    return 0;
}

注意数据类型：前缀和很容易超出 int 的范围。 如果数组元素个数是 10^5，每个元素最大 10^9，前缀和最大 10^14，必须用 long long。

5. 二维前缀和（拓展）

如果问题变成"子矩阵求和"，就需要二维前缀和。这是为后续数组DP和搜索做铺垫。

二维前缀和递推 
pre[i][j] = a[i][j] 
          + pre[i-1][j] 
          + pre[i][j-1] 
          - pre[i-1][j-1];

// 子矩阵 (x1,y1) 到 (x2,y2) 的和
sum = pre[x2][y2] 
    - pre[x1-1][y2] 
    - pre[x2][y1-1] 
    + pre[x1-1][y1-1];

二维前缀和的核心思想和一维完全一样：用容斥原理，把大矩形拆成几个已知前缀和的组合。

IOI教练常见教学陷阱

陷阱1：下标从0还是从1开始搞混

很多教材为了简化，把前缀和数组下标从1开始教。 但如果学生L1-L2习惯了0-based，突然改1-based会混乱。 建议统一用0-based，并通过 if (l==0) 处理边界。

陷阱2：前缀和数组用int导致溢出

这是竞赛中最高频的隐性Bug之一。从L3就要养成"求和先想 long long"的习惯。

陷阱3：只讲公式不讲思想

前缀和的灵魂是"空间换时间"和"预处理加速查询"。 如果只让学生背 pre[r]-pre[l-1]，他们遇到变体题就不会做了。

课后练习题

1

区间求和

入门

给定 n 个整数和 q 次查询，每次查询区间 [l, r] 的和。用前缀和 O(1) 回答。

2

最大连续子段和（入门版）

重点

给定 n 个整数，找出连续子数组的最大和。提示：枚举左端点，用前缀和快速计算。

3

子矩阵求和

重点

给定 n×m 矩阵和 q 次查询，每次查询子矩阵 (x1,y1) 到 (x2,y2) 的元素之和。

4

差分思想初步

挑战

给定长度为 n 的数组，进行 q 次操作：把区间 [l, r] 内每个数都加上 x。要求在 O(n+q) 内完成所有操作。

本课自测清单

  • 能独立写出前缀和预处理代码
  • 能正确应用区间求和公式 pre[r] - pre[l-1]
  • 知道前缀和数组要用 long long
  • 理解"预处理 O(n)，查询 O(1)"的时间优势
  • 能写出二维前缀和的递推公式（拓展）
  • 如果任何一条不满足，不要进入L3-05简单DP入门
