数组越界与内存

数组内存模型、越界危害、内存安全意识

8课时
约2周
难度

本课三维目标

知识 数组内存模型、越界概念、栈溢出、内存覆盖
技能 能识别常见的越界写法，养成边界检查习惯
思维 理解"数组大小"与"合法下标范围"的区别

数组越界是C++初学者最隐蔽、最致命的Bug。它不会每次都报错， 有时程序还能"正常"运行，但结果却是错的。这种Bug在比赛中特别可怕—— 你以为AC了，实际是RE（运行时错误）或者WA（答案错误）的随机抽奖。

1. 数组大小 vs 合法下标

这是初学者最容易混淆的概念：

越界示例 
int a[5]; // 数组大小是5

// 合法下标：0, 1, 2, 3, 4
a[0] = 10; // ✅ 合法
a[4] = 50; // ✅ 合法

// 非法下标
a[5] = 60; // ❌ 越界！数组只有5个元素，最大下标是4
a[-1] = 0; // ❌ 越界！下标不能为负数

关键记忆点：如果数组大小是 n，合法下标范围永远是 0 到 n-1。 循环写 i < n 而不是 i <= n。

2. 越界时到底发生了什么？

C++的数组不会自动检查边界。当你访问 a[5] 时， 编译器直接计算地址：起始地址 + 5 × 4，然后读写那片内存。

那片内存里可能是：

  • 其他变量的值 → 你的代码"碰巧"改掉了别的数据，程序行为诡异
  • 系统关键数据 → 程序直接崩溃（Segmentation Fault）
  • 未使用的内存 → 程序看似正常运行，但结果是错的

3. 常见越界错误写法

高危代码集锦 
// 错误1：循环条件写错
for (int i = 0; i <= n; i++) { ... a[i] ... } // 当 i==n 时越界

// 错误2：用1-based下标处理0-based数组
for (int i = 1; i <= n; i++) { ... a[i] ... } // a[n]越界

// 错误3：二维数组只检查了一个维度
if (x >= 0 && x < n) {
    a[x][y] = ...; // 如果 y 越界呢？
}

// 错误4：字符串末尾的 '\0'
char s[10] = "hello world"; // 字符串长度11，数组只有10

检查口诀：写数组访问前，默念三遍： "下标是否 ≥ 0？下标是否 < 数组大小？边界条件测试了吗？"

4. 竞赛中的防御性编程

在竞赛和实际工程中，推荐以下防御习惯：

防御性写法 
// 习惯1：数组开大一点
int a[1005]; // 题目说最多1000个，多开5个防手滑

// 习惯2：宏定义或常量管理数组大小
const int MAXN = 1005;
int a[MAXN];

// 习惯3：访问前检查边界
if (i >= 0 && i < MAXN) {
    a[i] = x;
}

// 习惯4：循环边界用 < 而不是 <=
for (int i = 0; i < n; i++) { ... }

5. 栈溢出：数组太大的后果

在函数内部定义的数组（局部数组）存放在栈区，栈的空间通常只有几MB。 如果你开了太大的数组：

栈溢出示例 
int main() {
    int a[10000000]; // 一千万个int = 40MB
    // 程序直接崩溃！栈空间不够
}

解决方案：把大数组定义为全局变量（放在函数外面），全局变量存放在数据区，空间远大于栈。

正确的全局数组写法 
const int MAXN = 1000005;
int a[MAXN]; // 全局数组，安全

int main() {
    // 正常使用
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }
}

IOI教练常见教学陷阱

陷阱1：认为"程序没报错就没越界"

C++不会帮你检查数组边界。很多越界程序看起来运行正常，但结果是错的。 要让学生建立"越界 = 定时炸弹"的意识。

陷阱2：不教全局数组的用法

很多教材为了追求"规范"，只讲局部数组。但竞赛中100%会用全局大数组。 L3就要让学生知道什么时候该用全局数组。

陷阱3：数组大小开得刚刚好

竞赛中数组多开10%~20%是标准操作。题目说1000就开1010或1050， 防止边界条件手滑或者题面描述有歧义。

找出越界代码

入门

2

修复越界Bug

重点

给定一段有数组越界问题的程序，要求学生找出并修复所有越界点。

3

安全的邻居访问

重点

给定 n×m 矩阵和中心点坐标 (x,y)，安全地输出其上下左右邻居的值（边界上的点没有某些邻居，不要越界）。

4

数组大小估算

挑战

题目要求处理 10^5 个整数，每个整数最大 10^9。计算需要开多大的数组，并选择全局还是局部定义。
