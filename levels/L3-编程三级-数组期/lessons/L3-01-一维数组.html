一维数组

数组声明、初始化、遍历、下标操作、经典应用

10课时
约2.5周
难度

本课三维目标

知识 数组定义、初始化、越界概念、for循环遍历数组
技能 能独立用数组存储并处理批量数据（如学生成绩）
思维 建立"下标思维"，理解数组与循环的天然配合关系

数组不是C++的语法点，而是从单点数据到批量数据的思维升级。

很多学生记住 int a[10]; 的写法，但遇到问题还是用10个变量。 这一课的核心不是语法，而是下标意识。

1. 为什么需要数组？

假设我们要存储100个学生的成绩。如果没有数组：

不用数组的写法 
int score1, score2, score3, score4, score5; // ... 写到 score100？
cin >> score1 >> score2 >> score3; // 输入 nightmare
int sum = score1 + score2 + score3; // 计算 nightmare

而有了数组，一切都变得自然：

用数组的写法 
int score[100];
for (int i = 0; i < 100; i++) {
    cin >> score[i];
}
int sum = 0;
for (int i = 0; i < 100; i++) {
    sum += score[i];
}

数组的本质：把一批同类型的数据，用连续的编号（下标）管理起来。

2. 数组的声明与初始化

数组基本操作 
// 声明：类型 数组名[元素个数];
int a[5];           // 5个整数，初始值不确定

// 初始化方式
int b[5] = {1, 2, 3, 4, 5};  // 全部初始化
int c[5] = {1, 2};           // 前两个赋值，其余为0
int d[] = {1, 2, 3};         // 编译器自动算大小为3

// 全部清零的常用写法
int e[100] = {0};            // 第一个显式为0，其余自动补0

注意：C++中数组大小必须是常量（编译期确定），不能用变量直接定义。 如果需要变长数组，在GESP考级中会涉及 vector，但基础阶段先掌握静态数组。

4. 下标思维：数组的灵魂

所谓"下标思维"，就是用编号来思考问题。例如：

  • 第 i 个学生的成绩 → score[i]
  • 数字 x 出现了几次 → cnt[x]++（桶计数思想）
  • 第 i 天走了多少步 → step[i]

教练视角：很多学生做题时，会写 a, b, c, d, e 分别代表周一到周五。 这本质上还是"变量思维"。真正学会数组的学生，会第一反应写 a[5] 并用下标表示星期几。

5. 经典应用：桶计数

桶计数是数组最经典的用法之一，也是数组DP线和搜索线的重要基础。

统计每个数字出现的次数 
int cnt[101] = {0}; // 假设数字范围是 0~100
int n;
cin >> n;
for (int i = 0; i < n; i++) {
    int x;
    cin >> x;
    cnt[x]++; // 用数字本身当下标！
}
for (int i = 0; i <= 100; i++) {
    if (cnt[i] > 0) {
        cout << i << " 出现了 " << cnt[i] << " 次" << endl;
    }
}

cnt[x]++ 这行代码，是无数高级算法的起点。

6. 遍历数组的常见模式

数组遍历四大模板 
// 1. 顺序遍历
for (int i = 0; i < n; i++) { ... a[i] ... }

// 2. 逆序遍历
for (int i = n - 1; i >= 0; i--) { ... a[i] ... }

// 3. 同时访问两个位置
for (int i = 0; i < n / 2; i++) {
    swap(a[i], a[n - 1 - i]); // 数组翻转
}

// 4. 滑动窗口（为后续课程做铺垫）
for (int i = 0; i < k; i++) { ... }
for (int i = k; i < n; i++) {
    // 移除 a[i-k]，加入 a[i]
}

IOI教练常见教学陷阱

陷阱1：死记硬背下标从0开始

学生容易背"下标从0开始"却不知道为什么。要告诉他们：下标代表的是偏移量， 第0个元素偏移0字节，第1个元素偏移4字节（int），这样计算机才能用加法直接算出地址。

陷阱2：数组和变量混用

很多题目明明可以一个数组解决，学生偏偏写5个变量。遇到这类情况要刻意追问： "如果数据变成100个，你的代码还写得完吗？"

陷阱3：一学数组就讲排序

排序算法（冒泡、选择）应该在L4分治线系统讲。L3的重点是下标意识和基础操作， 过早引入排序会分散注意力。

课后练习题

1

数组求和与平均值

入门

读入 n 个整数，计算它们的总和与平均值。

2

找最大值及下标

入门

读入 n 个整数，找出最大值以及它第一次出现的下标。

3

桶计数：统计票数

重点

n 个人投票，候选人编号 1~10。统计每个候选人得票数，输出票数最高者。

4

数组翻转

重点

读入 n 个整数，将其顺序翻转后输出。要求在原数组上操作，不能借助第二个数组。

  • 看到"统计出现次数"类题目，能第一反应想到桶计数
  • 能独立完成"数组翻转"代码
