简单DP入门

递推思想、数组存储状态、斐波那契、爬楼梯

12课时
约3周
难度

本课三维目标

知识 递推关系、状态定义、状态转移方程、初始条件
技能 能用数组递推解决斐波那契、爬楼梯、简单路径计数
思维 建立"大问题拆成小问题，小问题存数组复用"的DP思维

DP（动态规划）是很多学生竞赛生涯中的"鬼门关"。

但DP的起点一点都不神秘，它就是递推 + 数组存结果。 斐波那契数列本质上就是一个最简单的DP。

在L3引入简单DP，不是为了让学生立刻会做背包问题，而是为了在他们心中种下一颗种子： 数组不只是存数据，还可以存状态、存子问题的答案。

1. 从斐波那契看DP的本质

斐波那契数列：1, 1, 2, 3, 5, 8, 13... 每一项都是前两项之和。

斐波那契的递推写法 
long long f[100];
f[0] = 1;      // 初始条件
f[1] = 1;      // 初始条件
for (int i = 2; i < n; i++) {
    f[i] = f[i-1] + f[i-2];  // 状态转移方程
}
cout << f[n-1] << endl;

这里 f[i] 就是状态："第 i 个斐波那契数的值"。
f[i] = f[i-1] + f[i-2] 就是状态转移方程。
f[0]=1, f[1]=1 就是初始条件/边界。

核心发现：如果没有数组存储中间结果，而是用递归反复计算 fib(n-1) 和 fib(n-2)， 同样的问题会被计算无数次。数组的作用就是记忆化——把算过的答案存下来，下次直接查表。

2. 经典问题：爬楼梯

问题：有 n 级台阶，每次可以爬 1 级或 2 级，问有多少种不同的爬法？

思考方式：要爬到第 n 级，最后一步只能从第 n-1 级跨1步，或者从第 n-2 级跨2步。 所以：

爬楼梯的状态转移 
// f[i] = 爬到第 i 级台阶的方法数
f[0] = 1; // 地面，认为有1种方法（起点）
f[1] = 1; // 只有1级，只能跨1步

for (int i = 2; i <= n; i++) {
    f[i] = f[i-1] + f[i-2];
}

发现和斐波那契完全一样！很多看似不同的题目，背后的数学结构是相通的。 这就是学DP的价值——学会抽象状态、发现递推关系。

3. DP的三要素

每一个DP问题，都要明确回答三个问题：

  1. 状态是什么？
    用一个数组元素表示什么含义？（如 f[i] = 到第 i 级台阶的方法数）
  2. 状态转移方程是什么？
    当前状态如何从之前的状态推导出来？（如 f[i] = f[i-1] + f[i-2]）
  3. 初始条件/边界是什么？
    最小问题的答案是什么？（如 f[0]=1, f[1]=1）

教练口诀："定义状态、写转移方程、别忘了边界！" 这三句话从L3一直用到国家队选拔。

5. 空间优化：滚动数组

在爬楼梯问题中，计算 f[i] 只需要 f[i-1] 和 f[i-2]， 所以不需要保存整个数组。

空间优化到 O(1) 
long long a = 1, b = 1, c;
for (int i = 2; i <= n; i++) {
    c = a + b;  // c 就是 f[i]
    a = b;      // a 变成 f[i-1]
    b = c;      // b 变成 f[i]
}
cout << b << endl;

滚动数组是DP优化的基础技巧。在L3阶段，先让学生理解"为什么可以优化"， 不需要强求他们每次都写优化版本。

6. 拓展：路径计数

问题：在一个 n×m 的网格中，从左上角 (0,0) 走到右下角 (n-1,m-1)，每次只能向右或向下走，问有多少种不同的走法？

网格路径计数 
// f[i][j] = 走到 (i,j) 的路径数
// 只能从上方 (i-1,j) 或左方 (i,j-1) 过来
f[0][0] = 1;
for (int i = 0; i < n; i++) {
    for (int j = 0; j < m; j++) {
        if (i > 0) f[i][j] += f[i-1][j];
        if (j > 0) f[i][j] += f[i][j-1];
    }
}

这是二维DP的最简单形态，也是为后续背包、区间DP打基础。

IOI教练常见教学陷阱

陷阱1：一开口就讲"动态规划"四个字

"动态规划"这个术语对初学者很有压迫感。L3阶段不要强调这个词， 而是说"递推"、"用数组存答案"。等学生习惯了递推，再告诉他们这就是DP。

陷阱2：只给模板不给推导过程

DP最难的是"怎么想到状态定义和转移方程"。 教学中要放慢速度，从具体例子出发，让学生自己发现递推关系。

陷阱3：跳过边界条件

很多DP代码错误都出在边界条件上。要反复强调：写转移方程之前，先把最小的情况手动算出来。

斐波那契数列

用数组递推求第 n 个斐波那契数。n ≤ 50。

2

爬楼梯

重点

n 级台阶，每次可以爬 1 级或 2 级，求爬到顶的方法数。

3

跳台阶变体

重点

n 级台阶，每次可以爬 1 级、2 级或 3 级，求爬到顶的方法数。

4

网格路径计数

挑战

在 n×m 网格中，从左上角走到右下角，只能向右或向下，求路径总数。
