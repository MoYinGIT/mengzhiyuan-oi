线段树与树状数组

单点修改区间查询、区间修改、Lazy Propagation、BIT基础

10课时
约2.5周
难度

本课三维目标

知识 线段树的结构、树状数组的原理、Lazy Tag、lowbit运算
技能 能独立写出线段树和树状数组的单点修改+区间查询代码
思维 理解"分治维护区间信息"的高级数据结构思想

1. 为什么需要线段树和树状数组？

假设有一个数组，需要频繁进行两种操作：

  • 修改某个元素的值
  • 查询某个区间的和/最大值

如果用朴素方法：修改是 O(1)，但区间查询是 O(n)。 如果用前缀和：查询是 O(1)，但修改后需要重新计算前缀和，是 O(n)。

线段树和树状数组可以在 O(log n) 时间内同时完成修改和查询， 是处理动态区间问题的利器。

2. 线段树的结构

线段树是一棵二叉树，每个节点代表一个区间。 根节点代表整个区间 [1, n]，叶子节点代表单个元素。 每个内部节点的值等于其两个子节点值的合并（如求和、求最大值）。

3. 线段树模板：单点修改 + 区间查询

线段树基础模板 
const int MAXN = 100005;
long long tree[MAXN * 4];
int a[MAXN];

// 建树
void build(int node, int l, int r) {
    if (l == r) {
        tree[node] = a[l];
        return;
    }
    int mid = (l + r) / 2;
    build(node * 2, l, mid);
    build(node * 2 + 1, mid + 1, r);
    tree[node] = tree[node * 2] + tree[node * 2 + 1];
}

// 单点修改
void update(int node, int l, int r, int idx, int val) {
    if (l == r) {
        tree[node] = val;
        return;
    }
    int mid = (l + r) / 2;
    if (idx <= mid) update(node * 2, l, mid, idx, val);
    else update(node * 2 + 1, mid + 1, r, idx, val);
    tree[node] = tree[node * 2] + tree[node * 2 + 1];
}

// 区间查询
long long query(int node, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return tree[node];
    int mid = (l + r) / 2;
    long long sum = 0;
    if (ql <= mid) sum += query(node * 2, l, mid, ql, qr);
    if (qr > mid) sum += query(node * 2 + 1, mid + 1, r, ql, qr);
    return sum;
}

线段树的数组通常开原数组的4倍大小（MAXN * 4），因为完全二叉树的最坏情况需要这么多节点。

4. 树状数组（BIT / Fenwick Tree）

树状数组比线段树更简洁，空间更小，代码更短。 它的核心是利用 lowbit(x) = x & (-x) 来维护前缀信息。

树状数组模板 
long long bit[MAXN];
int n;

int lowbit(int x) {
    return x & (-x);
}

// 在位置idx加上val
void add(int idx, long long val) {
    for (int i = idx; i <= n; i += lowbit(i)) {
        bit[i] += val;
    }
}

// 求前缀和 [1, idx]
long long query(int idx) {
    long long sum = 0;
    for (int i = idx; i > 0; i -= lowbit(i)) {
        sum += bit[i];
    }
    return sum;
}

// 区间查询 [l, r]
long long rangeQuery(int l, int r) {
    return query(r) - query(l - 1);
}

lowbit 的作用是找到 x 的最低位1所代表的数值。 树状数组通过不断加减 lowbit，巧妙地覆盖了所有需要更新的节点。

5. 线段树 vs 树状数组

特性 线段树 树状数组
代码长度 较长 很短
功能范围 支持任意满足结合律的操作 主要支持求和、计数
区间修改 支持（Lazy Tag） 有限支持（差分思想）
竞赛建议 通用、功能强大 求和场景首选，代码不易出错

6. Lazy Propagation（拓展）

当问题变成"区间修改 + 区间查询"时，单点修改的线段树不够用了。 需要引入 Lazy Tag（懒惰标记）： 当修改一个完整区间时，不立即更新子节点，而是打一个标记，等后续查询到该子节点时再下放标记。

Lazy Tag核心思想 
long long lazy[MAXN * 4];

// 下放标记
void pushDown(int node, int l, int r) {
    if (lazy[node] != 0) {
        int mid = (l + r) / 2;
        lazy[node * 2] += lazy[node];
        lazy[node * 2 + 1] += lazy[node];
        tree[node * 2] += lazy[node] * (mid - l + 1);
        tree[node * 2 + 1] += lazy[node] * (r - mid);
        lazy[node] = 0;
    }
}

Lazy Tag是线段树的精髓，也是很多高级数据结构（如树链剖分、可持久化线段树）的基础。

IOI教练常见教学陷阱

陷阱1：线段树数组开太小

线段树数组必须开到 4*n。很多学生在比赛中因为这个开2*n或3*n而RE（运行时错误）， 这是非常可惜的丢分。

陷阱2：树状数组下标从0开始

lowbit(0) = 0，会导致树状数组的死循环。树状数组的下标必须从1开始。

陷阱3： Lazy Tag忘记下放

写Lazy Tag时，每次query和update进入子节点之前，都必须先pushDown。 漏掉一次，整个区间的值就会出错。
