状态压缩与计数DP

状态压缩DP、计数DP、容斥原理、组合数取模

10课时
约2.5周
难度

本课三维目标

知识 二进制状态压缩、子集枚举、计数DP转移、容斥原理、Lucas定理入门
技能 能用状态压缩DP解决n≤20的排列/选择问题，能写基础计数DP

1. 状态压缩：用二进制表示集合

当问题规模很小（通常 n ≤ 20）时，我们可以用一个整数的二进制位来表示一个集合的状态。 整数中的第 i 位为 1，表示第 i 个元素被选中（或已访问）。

二进制状态操作 
int state = 0;

// 将第i位置为1
state |= (1 << i);

// 将第i位置为0
state &= ~(1 << i);

// 判断第i位是否为1
if (state & (1 << i)) { ... }

// 枚举state的所有子集
for (int sub = state; sub; sub = (sub - 1) & state) {
    // sub 是 state 的一个非空子集
}

例如，state = 5 对应的二进制是 101， 表示第0个和第2个元素被选中了。

2. 旅行商问题（TSP）：状态压缩DP经典

问题：给定 n 个城市和它们之间的距离，从城市0出发，访问所有城市恰好一次后回到起点， 求最短路径长度。

状态定义：dp[state][i] 表示已经访问了 state 中的城市， 且当前位于城市 i 时的最短路径长度。

const int INF = 1e9;
int dist[20][20];
int dp[1 << 20][20];

int n;

int solve() {
    for (int i = 0; i < (1 << n); i++)
        for (int j = 0; j < n; j++)
            dp[i][j] = INF;

        for (int i = 0; i < n; i++) {
            if (!(state & (1 << i))) continue;  // i不在state中，跳过

3. 子集枚举的应用

状态压缩DP中，经常需要枚举一个集合的所有子集。例如： 把一群人分成两组，求某种最优分配。

枚举所有子集 
int n = 5;
int full = (1 << n) - 1;  // 11111

// 枚举full的所有子集
for (int sub = full; sub >= 0; sub = (sub - 1) & full) {
    // sub 是当前枚举到的子集
    if (sub == 0) break;  // 注意：0也是子集，根据需要处理
}

// 如果只枚举非空真子集
for (int sub = full; sub; sub = (sub - 1) & full) {
    int other = full ^ sub;  // 补集
    // ...
}

sub = (sub - 1) & full 是枚举子集的标准技巧， 它保证按从大到小的顺序枚举 full 的所有子集。

4. 计数DP与组合数

计数DP的目标不是求最大/最小值，而是求方案数。 计数DP的转移通常用加法而不是取max/min。

网格路径计数 
// dp[i][j] = 从(0,0)走到(i,j)的路径数
const int MOD = 1e9 + 7;

dp[0][0] = 1;
for (int i = 0; i < n; i++) {
    for (int j = 0; j < m; j++) {
        if (i > 0) dp[i][j] = (dp[i][j] + dp[i-1][j]) % MOD;
        if (j > 0) dp[i][j] = (dp[i][j] + dp[i][j-1]) % MOD;
    }
}

组合数可以用递推公式预处理： C(n, m) = C(n-1, m-1) + C(n-1, m)

组合数预处理 
const int MAXN = 1005;
const int MOD = 1e9 + 7;
long long C[MAXN][MAXN];

void initComb() {
    for (int i = 0; i < MAXN; i++) {
        C[i][0] = C[i][i] = 1;
        for (int j = 1; j < i; j++) {
            C[i][j] = (C[i-1][j-1] + C[i-1][j]) % MOD;
        }
    }
}

5. 容斥原理入门

容斥原理用于计算"满足至少一个条件"的方案数，通过加减交集来避免重复计数。

容斥原理：错位排列 
// 求1~n的排列中，没有任何一个数在原来位置上的排列数（错位排列）
// 用容斥：总排列 - 至少1个在原位 + 至少2个在原位 - ...

long long derangement(int n) {
    long long ans = 0;
    for (int i = 0; i <= n; i++) {
        long long term = C[n][i] * fact[n - i] % MOD;
        if (i % 2 == 0) ans = (ans + term) % MOD;
        else ans = (ans - term + MOD) % MOD;
    }
    return ans;
}

IOI教练常见教学陷阱

陷阱1：状态压缩时数组开不够

n=20时，状态数是 2^20 = 1,048,576。dp数组要开到 [1<<20][20]， 约2000万个元素。如果用int，需要约80MB，要注意空间限制。

陷阱2：计数DP忘记取模

方案数通常非常大，每一步转移都要取模。如果忘了取模，整数溢出会导致WA。

陷阱3：子集枚举遗漏空集

for (int sub = state; sub; sub = (sub-1) & state) 不会枚举空集。如果需要空集，要单独处理。
