# GLM5 提示词：生成 铁律一：考点不等于教学目标.html

## 角色设定

你是全球最顶级的 IOI 教练，曾带领多名学生在国际信息学奥林匹克竞赛中获得金牌。

你的教学理念：
1. 【考点≠教学目标】考指针不是教指针，是教内存地址概念
2. 【能力主线优先】递归线必须三级启动，前缀和必须和一维数组同步教
3. 【三遍学习法】裸用→组合→变体，跳过任何一遍学生只会背代码
4. 【禁止先背再理解】排序从"为什么排"引入，代码永远是最后一步
5. 【基础不能跳】不理解内存分层就理解不了局部变量消失
6. 【检验标准>教学内容】能独立解决变体题才算真会

你的教学风格：
- 用生动的比喻解释抽象概念（酒店房间=内存、俄罗斯套娃=递归）
- 强调"为什么"而不是"是什么"
- 从数百名学生的血泪教训中总结教学陷阱
- 既有严格的标准，又有对学生的鼓励和期望
- 使用第一人称"我从学生身上看到...""我的经验是..."

### 3. 核心概念讲解（重点板块）

**标题**：`什么是"考点≠教学目标"？`

**内容要求**（IOI教练视角）：

#### 3.1 一个真实的教学场景
用第一人称描述：
> "我曾经教过一个学生，小明（化名）。他四级考试前，我让他写一段代码：
> ```cpp
> int a = 10;
> int *p = &a;
> cout << *p;
> ```
> 他能完美写对，指针的语法他背得很熟。但我问他：'p 里面存的是什么？'他说：'p 存的是 a。'
> 
> 问题在哪里？他不理解 p 里面存的是 **a 的地址**，而不是 a 本身。他只是记住了`*p`这种写法。
> 
> 后来学五级链表时，他彻底崩溃了。因为链表需要理解'指针指向的是下一个节点的地址'，而他根本没有'地址'这个概念。"

#### 3.2 考点 vs 教学目标 对比表

| 维度 | 考点（考试要求） | 教学目标（能力终点） |
|------|-----------------|---------------------|
| **关注点** | 学生能不能写出语法正确的代码 | 学生是否理解背后的概念 |
| **检验方式** | 看代码能否通过编译、输出正确 | 看学生能否解释"为什么"、能否解决变体题 |
| **短期效果** | 快，背模板就能过 | 慢，需要理解内化 |
| **长期效果** | 五级、六级时知识断层 | 知识体系完整，能举一反三 |
| **例子：指针** | 会写 `int *p = &a;` | 理解"p 里面存的是地址，不是值" |
| **例子：递归** | 能默写递归函数模板 | 理解"递归是问题的自我相似性" |
| **例子：排序** | 会调用 `sort()` | 理解"为什么这样排"、能手写排序过程 |

#### 3.3 IOI教练的核心观点

> "GESP 考试就像体检。体检报告告诉你'血压偏高'（考点），但治疗高血压需要理解'血压是什么''为什么会高''怎么调节'（教学目标）。
> 
> 如果只教'怎么写出通过考试的代码'，就像只教学生'怎么在体检表上填正常数值'，而不是真正治好病。"

**样式要求**：

- 对比表使用交替行背景色
- IOI教练观点使用特殊背景（金色淡化）

### 4. 反面教材 vs 正面教材（对比板块）

**标题**：`❌ 反面教材 vs ✅ 正面教材`

**左侧（反面-红色系）**：
- 标题：直接教指针语法
- 教学场景：
  - 老师："今天我们学指针。`*p` 是取值，`&a` 是取地址。记住这个公式。"
  - 学生：记下来 `int *p = &a; cout << *p;`
  - 结果：学生能应付四级考试，但五级链表完全不懂
- 学生几个月后的反馈：
  > "老师，我觉得四级和五级之间有个鸿沟，我完全跳不过去。"

**右侧（正面-绿色系）**：
- 标题：从内存模型引入地址概念
- 教学场景（IOI教练的做法）：
  - 先花2课时讲内存：变量是内存中的一块空间，每个空间有门牌号（地址）
  - 用酒店房间比喻：变量名是房间标签，地址是门牌号，指针是"记录门牌号的便签"
  - 再引入语法：`*p` 是"去这个门牌号的房间取东西"
  - 结果：学生理解"地址"概念，五级链表时只需要说"next指针存的是下一个节点的地址"
- 学生几个月后的反馈：
  > "老师，我觉得五级只是四级的延伸，很自然。"

**样式要求**：
- 两列对比布局（移动端单列）
- 左侧背景：`var(--red-dim)`，边框红色
- 右侧背景：`var(--green-dim)`，边框绿色
- 使用引用块展示学生反馈

### 5. 为什么"考什么教什么"是错的？（深度分析）

**标题**：`为什么家长和机构容易陷入"考什么教什么"的误区？`

**内容要求**：

#### 5.1 误区一：应试教育的惯性
> "我们的教育体系中，'考什么教什么'是常态。高考考什么，高中就教什么。这导致家长和学生形成了条件反射：考试范围=学习范围。
> 
> 但竞赛编程不一样。GESP四级考指针语法，但指针语法的**前置知识**是内存地址概念。只教语法不教概念，就像只教'怎么按电梯按钮'不教'电梯为什么能上下'。"

#### 5.2 误区二：短期见效 vs 长期能力
> "教语法见效快。学生一周就能写出指针代码，家长看到成果很高兴。但教概念见效慢，需要2-3周才能建立内存模型。
> 
> 问题在于：四级到五级只有3个月间隔。如果四级只教了语法，五级时学生需要同时补'地址概念'和学'链表'，时间根本不够。
> 
> 我的做法是在四级就把'地址概念'教透，五级链表只需要2课时就能讲明白。"

#### 5.3 误区三：把"通过考试"当成"学会编程"
> "GESP考试是水平测试，不是能力认证。通过四级只代表你掌握了四级考点的**最小子集**，不代表你具备了四级对应的**编程能力**。
> 
> 我见过太多学生：GESP四级90分，但给一道需要灵活运用指针的题就完全不会。因为他们学的是'考试技巧'，不是'编程思维'。"

### 6. 如何确定真正的教学目标？（方法论）

**标题**：`IOI教练的方法：如何确定真正的教学目标？`

**内容要求**：

#### 6.1 问自己三个问题
1. **概念前置**：这个考点的**前置概念**是什么？
   - 例：指针的前置是"内存地址"，递归的前置是"自我相似性"
2. **知识迁移**：学生学会这个后，能否**自然延伸**到下一个级别？
   - 例：学会"地址"后，链表、函数传指针、数组名本质都很自然
3. **变体解决**：给一道**没见过的题**，学生能不能用学到的知识解决？
   - 例：背过`int *p = &a;`的学生，遇到`int **pp = &p;`（二级指针）就会懵

#### 6.2 以"指针"为例的教学目标拆解

| 级别 | 表面考点 | 真正的教学目标 | 检验方式 |
|------|---------|---------------|---------|
| L1 | 变量 | 理解"变量是内存空间" | 能画出`int a=5;`的内存图 |
| L2 | 内存地址（了解） | 理解"每个变量有地址" | 能解释`&a`是什么 |
| L3 | 数组 | 理解"数组名是首元素地址" | 能用指针遍历数组 |
| L4 | 指针语法 | 理解"指针存的是地址" | 能解释`p`和`*p`的区别 |
| L5 | 链表 | 理解"指针连接节点" | 能手画链表内存图 |

> "注意：我在L1就开始铺垫'内存'概念，虽然L1不考指针。这样到L4时，学生已经建立了3个月的内存直觉，指针只是'给这个直觉起个名字'。"

### 7. 给家长和老师的建议（行动指南）

**标题**：`💡 给家长和老师的具体建议`

**内容要求**：

#### 对家长：
1. **问老师**：不要问"孩子什么时候能过四级"，要问"孩子是否理解了内存地址概念"
2. **看作业**：不要只看代码能否运行，要让孩子**画内存图**解释
3. **耐心**：理解概念比背语法慢2-3倍，但这是值得的投资

#### 对老师：
1. **反向设计**：从五级链表倒推四级需要准备什么前置知识
2. **概念先行**：每教一个新语法，先讲"这是什么概念"
3. **变体检验**：每节课留一道"课本上没有"的变式题

### 8. 教学陷阱警示（红色警示框）

**标题**：`⚠️ 教学陷阱：考纲导向的教学`

**内容**：
- **陷阱描述**：严格按照GESP考纲的"知识点清单"教学，考什么讲什么，不考的完全不提
- **后果**：学生知识体系碎片化，无法形成能力主线，五级六级时大量补窟窿
- **我的血泪教训**：
  > "我曾经严格按照考纲教，结果20个学生里有8个在四级到五级之间流失。他们不是因为不努力，是因为'听不懂了'。后来我反思，不是学生的问题，是我的教学没有为他们搭建好认知阶梯。"
- **解决方法**：
  1. 看到考纲里的"指针"，要想"学生需要先理解什么"
  2. 主动补充考纲没写但必要的前置知识（如L1的内存概念）
  3. 每个知识点问："学生到五级还需要这个概念的什么延伸？"

**样式要求**：
- 红色背景（`--red-dim`）
- 红色边框
- 警示图标
- IOI教练第一人称叙述

### 9. 检验标准（验收清单）

**标题**：`✅ 检验标准：学生是否真正理解了"考点≠教学目标"？`

**内容**：给家长和老师的检验清单

- [ ] 孩子能解释"为什么学指针"（不是因为考试考，是因为要理解内存地址）
- [ ] 孩子能画出`int *p = &a;`的内存示意图
- [ ] 孩子能用自己的话解释"p 里面存的是什么"（答案是"地址"不是"10"）
- [ ] 遇到一道没见过的指针题，孩子能尝试用"地址"概念分析，而不是说"我没学过"
- [ ] 孩子知道"GESP四级只是检验手段，不是学习目标"

**拦路虎**：
> 如果孩子仍然认为"考试不考的内容就不用学"，说明他没有理解这条铁律。需要停下来，重新讨论"为什么学编程"。
