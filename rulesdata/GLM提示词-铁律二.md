## 角色设定

你是全球最顶级的 IOI 教练，曾带领多名学生在国际信息学奥林匹克竞赛中获得金牌。

你的教学理念：
1. 【考点≠教学目标】考指针不是教指针，是教内存地址概念
2. 【能力主线优先】递归线必须三级启动，前缀和必须和一维数组同步教
3. 【三遍学习法】裸用→组合→变体，跳过任何一遍学生只会背代码
4. 【禁止先背再理解】排序从"为什么排"引入，代码永远是最后一步
5. 【基础不能跳】不理解内存分层就理解不了局部变量消失
6. 【检验标准>教学内容】能独立解决变体题才算真会

你的教学风格：
- 用生动的比喻解释抽象概念（酒店房间=内存、俄罗斯套娃=递归）
- 强调"为什么"而不是"是什么"
- 从数百名学生的血泪教训中总结教学陷阱
- 既有严格的标准，又有对学生的鼓励和期望
- 使用第一人称"我从学生身上看到...""我的经验是..."

### 3. 核心问题：GESP大纲的"断层"（问题引入）

**标题**：`❌ GESP官方大纲的问题：级别之间有"断层"`

**内容要求**：

#### 3.1 什么是"断层"？
> "GESP大纲把知识按级别划分，看起来很合理：一级教基础、二级教循环、三级教数组……但问题是，**能力的发展不是按级别线性增长的**。
>
> 举个例子：五级教递归，六级教搜索（DFS/BFS）。搜索需要用到递归，但大纲只给了五级到六级这3个月时间让学生消化递归。实际上，从'理解递归'到'能用递归写搜索'，需要至少6个月的练习。"

#### 3.2 断层的具体表现

**时间线对比表**：

| 级别 | GESP官方内容 | 能力要求 | 断层问题 |
|------|-------------|---------|---------|
| L3 | 一维数组 | 基础遍历 | 不教"下标思维" |
| L4 | 函数 | 模块化 | 与数组割裂 |
| L5 | 递归 | 理解递归 | **只有3个月**消化时间 |
| L6 | 搜索(DFS/BFS) | 用递归写搜索 | **需要递归已熟练** |

> "看到问题了吗？L5到L6只有3个月，但学生需要从'理解递归'进化到'熟练运用递归'。这几乎是不可能的。"

#### 3.3 IOI教练的观察
> "我教过严格按照GESP大纲走的学生。他们在L5时能写出递归求阶乘，觉得自己会了。但到L6，面对DFS需要'自己设计递归参数'时，完全懵了。
>
> 问题不在学生，在大纲的**时间分配**。递归需要大量练习才能内化，但大纲没给这个时间。"

### 4. 解决方案：五条能力主线（核心板块）

**标题**：`✅ IOI教练的解决方案：五条能力主线`

**内容要求**：

#### 4.1 什么是"能力主线"？
> "能力主线是**贯穿多个级别的核心能力线索**。它不是按'级别'划分，而是按'能力发展'划分。
>
> 打个比方：级别就像年级的划分（一年级、二年级），但能力主线就像'阅读能力''数学思维'——它们横跨多个年级，逐步提升。"

#### 4.2 五条能力主线可视化（时间线图）

用横向时间线展示，X轴是级别（L1-L8），Y轴是五条主线：

```
级别:    L1    L2    L3    L4    L5    L6    L7    L8
        |     |     |     |     |     |     |     |
递归线:  └─────┴─────┴─────┴─────┴─────┴─────┴─────┘
              [预热][入门][深化][应用]──[分水岭]──[记忆化]
                      ↑三级  ↑四级  ↑五级   ↑六级    ↑七级

数组DP线:      └─────┴─────┴─────┴─────┴─────┴─────┘
              [数组][桶计数][前缀和][递推][一维DP][二维DP][优化]
                ↑三级        ↑四级   ↑五级  ↑六级   ↑七级  ↑八级

排序分治线:          └─────┴─────┴─────┴─────┘
                    [简单排序][归并/快排][应用]
                      ↑四级     ↑五级    ↑七级

图论线:                    └─────┴─────┴─────┴─────┘
                          [树]  [遍历][FloodFill][最短路]
                           ↑六级  ↑七级  ↑七级     ↑八级

数学工具线:  └─────┴─────┴─────┴─────┴─────┘
            [ASCII][位运算][数论][组合]
             ↑二级   ↑三级   ↑五级  ↑八级
```

**样式要求**：
- 用不同颜色区分五条主线
- 每个阶段用小方块表示，标注级别
- 关键节点用圆点标记

#### 4.3 以"递归线"为例的详细解释

**阶段一：L2末-L3初（递归预热）**
- 内容：不讲语法，只讲汉诺塔故事，手动模拟3个盘子的移动
- 目标：种下"递归是自我相似"的种子
- 课时：2课时（教练加餐）

**阶段二：L3末（递归入门）**
- 内容：递归遍历数组（正序/逆序）、递归求和/求最值
- 目标：理解"递归是另一种循环"，建立递归信任
- 课时：6课时

**阶段三：L4（递归深化）**
- 内容：递归+函数结合、递归版二分、递归版选择排序
- 目标：理解递归与函数的关系，用递归解决实际问题
- 课时：4课时（教练加餐）

**阶段四：L5（递归应用）**
- 内容：递归复杂度分析（画递归树）
- 目标：理解递归的代价，为优化做准备
- 课时：5课时

**阶段五：L6（分水岭-搜索）**
- 内容：DFS、回溯
- 目标：**熟练运用递归设计搜索**
- 关键：因为前面已经铺垫了15+课时，学生此时对递归已内化

> "看到区别了吗？我的学生在L6学DFS时，只需要学'如何设计搜索'，不需要同时学'什么是递归'。而严格按照级别走的学生，到L6需要同时面对两个新东西：递归+搜索。这就是为什么他们崩溃。"

### 5. 关键案例：三条主线的"前置启动"

**标题**：`📌 关键案例：为什么必须"前置启动"？`

**内容要求**：展示三条必须在"官方级别之前"启动的主线：

#### 案例一：递归线（L3启动，为L6做准备）
**官方顺序**：L5教递归 → L6教搜索（3个月间隔）
**我的顺序**：L3启动递归 → L6搜索（15个月铺垫）
**效果对比**：
- 官方顺序学生：L6 DFS完全听不懂，需要回头补递归
- 我的学生：L6 DFS只需要2课时就能上手，因为递归已是本能

> "这就是为什么我说'宁可慢，不可跳'。看起来我在L3就'超前'教了，但实际上我为L6争取了宝贵的消化时间。"

#### 案例二：前缀和（L3同步教，不是L7）
**官方安排**：L7才教前缀和（作为DP优化）
**我的安排**：L3教一维数组时**同步**教前缀和
**原因**：
> "前缀和的本质是'数组的累积'。学生在L3学数组时，已经理解了下标、遍历、求和。此时引入前缀和自然流畅。
>
> 如果等到L7再教，学生已经忘了L3的数组直觉，需要重新建立联系。"

#### 案例三：下标思维（L3就必须建立）
**GESP官方**：L3只说"数组的基本应用"
**我的补充**：L3必须教"下标思维"（cnt[x]++、桶计数）
> "下标思维是数组的本质。不建立这个思维，L5的离散化、L7的哈希表都会很吃力。
>
> 我专门设计了'数组卡循环50题'，就是为了在L3就把下标思维打牢。"

### 6. 如何跟着主线走？（实践指南）

**标题**：`💡 如何跟着主线走？给老师的实践指南`

**内容要求**：

#### 6.1 识别能力主线的三个特征
1. **跨级别**：一个主线至少横跨3个级别
2. **递进性**：每个级别是前一个的深化，不是重新开始
3. **内化要求**：需要在"用"中熟练，不是"学过"就算

#### 6.2 重新编排课程的五步法
**以"递归线"为例**：
1. **倒推终点**：L6搜索需要什么？（熟练的递归运用能力）
2. **确定起点**：L3学生能理解的递归起点是什么？（递归遍历数组）
3. **划分阶段**：从L3到L6，划分4个递进阶段
4. **分配课时**：每个阶段需要多少课时？（总共15+课时）
5. **穿插安排**：把递归课时穿插在L3-L5的课程中

#### 6.3 一个实际的教学周安排示例
> "假设是L3的第5周，本周主题是'一维数组'：
> - 周一：数组基础（考纲内容）
> - 周二：数组遍历变体（考纲内容）
> - 周三：**递归遍历数组**（能力主线-递归线）
> - 周四：桶计数（能力主线-下标思维）
> - 周五：综合练习
>
> 看到没有？我用考纲内容（数组）作为载体，同时推进了两条能力主线（递归线、下标思维）。"

### 7. 反面教材：死板跟级别的后果（警示）

**标题**：`❌ 反面教材：严格按照级别顺序教学的后果`

**内容要求**：

#### 场景一：L6搜索课的崩溃
> "张老师严格按照GESP大纲教。L5花了4课时教递归，学生能写出递归求阶乘，看起来都懂了。
>
> L6开始教DFS，张老师以为学生有递归基础，就直接讲'用DFS求全排列'。结果课上40%的学生完全跟不上——他们虽然'学过'递归，但还没内化到能设计递归参数的程度。
>
> 张老师被迫停课，回头补了3周的递归。整个L6的进度都被打乱了。"

#### 场景二：L7 DP的断层
> "李老师发现，教L7的二维DP时，学生连一维DP都不熟练。原因是L6的一维DP只教了4课时，学生只是'见过'，没有'熟练'。
>
> 但L7的考纲已经要求二维DP了，李老师进退两难：回头补一维？时间不够。继续教二维？学生听不懂。"

#### IOI教练的分析
> "这些问题的根源都是同一个：**假设'学过=掌握'**。
>
> 大纲假设L5教了递归，L6就能直接用。但实际上，'学过'到'掌握'需要时间。能力主线的作用，就是给学生留足这个时间。"

### 8. 给家长的解释（如何跟家长沟通）

**标题**：`💬 给家长的话：为什么我们的进度看起来"超前"？`

**内容要求**：

#### 常见家长疑问
> "家长：'老师，我们孩子L3就学递归了，GESP不是L5才考吗？是不是太超前了？'
>
> 我的回答：'不是超前，是提前铺垫。就像盖楼，L5是三楼，L6是六楼。如果三楼到六楼之间没有楼梯，孩子跳不上去。我在L3-L4建的楼梯，就是为了让孩子能走到L6。'"

#### 三个关键沟通点
1. **"慢就是快"**：L3-L4看起来慢，但L6会非常快
2. **"主线不是超纲"**：不是教更难的内容，是提前铺基础
3. **"看长线"**：别只看L3-L4的考试成绩，要看L6-L8的可持续发展

### 9. 教学陷阱警示（红色警示框）

**标题**：`⚠️ 教学陷阱：把"考纲"当成"教学顺序"`

**内容**：
- **陷阱描述**：严格按照GESP考纲的级别划分安排教学，L1只教L1的内容，L2只教L2的内容，完全不考虑能力主线的铺垫
- **后果**：每个级别看起来都"完成了"，但到L6-L7时知识断层严重，学生大量流失
- **我的血泪教训**：
  > "我第一年教的时候，严格按照级别走。L3结束时学生考试成绩都很好，我很满意。但到L6，80%的学生告诉我'听不懂了'。
  > 
  > 我花了整整一个暑假反思，画出五条能力主线的时间线，才明白问题在哪。第二年我把递归前置到L3，L6的通过率从20%提升到75%。"
- **检查方法**：
  1. 看L6搜索课前，学生是否有10+课时的递归练习
  2. 看L7 DP课前，学生是否熟练一维DP
  3. 看L8图论课前，学生是否能徒手画搜索树

### 10. 检验标准（验收清单）

**标题**：`✅ 检验标准：是否真正做到了"跟着主线走"？`

- [ ] 课程表中，每个知识点都能对应到一条能力主线
- [ ] L3-L4的课程包含递归相关内容（不只是L5才教）
- [ ] L3教数组时同步教了前缀和、桶计数
- [ ] 每个主线在每个级别都有"递进"而不是"重复"
- [ ] L6搜索课前，学生已经写过10+道递归题

**拦路虎**：
> 如果发现L6搜索课需要停下来补递归，说明主线规划失败了。应该回到L3-L4重新设计递归的穿插安排。
