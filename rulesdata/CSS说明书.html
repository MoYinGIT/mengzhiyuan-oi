7 个 CSS 文件，各管各的事

每个文件只负责一类职责。改颜色不动布局，改布局不动组件，改组件不动级别色。 任何一个需求，只打开一个文件就能完成。

base.css layout.css components.css level.css philosophy.css mainline.css responsive.css
加载顺序即层叠顺序，后面的覆盖前面的。responsive 必须最后。

安全边界

改A的时候不需要看B。每个"不需要碰"都是一个安全边界，边界越多，改错东西的概率越低。

单向依赖

base ← layout ← components ← level/philosophy/mainline ← responsive。每个文件只依赖排在自己前面的文件，不存在循环依赖。

修改频率对齐

改频率相同的东西放同一个文件。base和layout几乎不改，components偶尔改，responsive测手机时改。频率不同，文件不同。

按需加载

做教案页只加载base+layout+components+level，不加载philosophy和mainline。不相关的代码不进入页面，找东西时少一半噪音。

01
base.css
全站的地基。不改样式，只定义规则。
修改频率：几乎不改
被依赖：所有其他6个文件
依赖：无
管什么
  • :root全站 CSS 变量的唯一真相源——颜色、间距、字号、圆角、字体、过渡
  • reset清掉浏览器默认 margin/padding，统一下盒模型
  • body全局字体、行高、背景色、最小高度
  • html滚动行为、滚动条样式
不管什么
  • 不定义任何布局（导航栏、侧边栏、grid）
  • 不定义任何组件（按钮、卡片、标签）
  • 不写任何具体颜色值在组件上（只定义变量）
  • 不写媒体查询
什么时候打开
  • 换全站字体
  • 换主色（改 --accent 一个值，全站跟着变）
  • 调整全局间距体系（比如把基准从8px改成6px）
  • 加新的全局变量
--bg-deep
--fg
--accent
--red
--green
--cyan
--border
--level3

这些变量定义在这里，其他6个文件永远不出现具体颜色值，只引用变量名。

02
layout.css
管"东西放在哪"。导航栏、侧边栏、内容区、页脚的位置和尺寸。
修改频率：几乎不改
被依赖：components / level / philosophy / mainline / responsive
依赖：base.css
管什么
  • .top-bar顶部导航栏——高度56px、粘性定位、毛玻璃背景
  • .content-with-sidebar有侧边栏布局——左260px右自适应的grid
  • .content-fullwidth全宽布局——目录总览、汇总页
  • .content-narrow窄居中布局——铁律详情页，680px
  • .sidebar侧边栏——sticky定位、滚动条、最大高度
  • .page-header页面头部——面包屑、标题、元信息行的结构
  • .page-nav页面底部导航——上一课/下一课双列grid
  • .site-footer页脚
  • .container通用容器——最大宽度1100px水平居中
  • .page-body页面主体排版——h2/h3/p/pre/code的基础样式
不管什么
  • 不管按钮长什么样
  • 不管卡片圆角和阴影
  • 不管颜色（只引用base的变量）
  • 不管级别色
  • 不管板块专属组件
什么时候打开
  • 导航栏高度要改（比如从56px改成48px）
  • 侧边栏宽度要改（比如从260px改成240px）
  • 内容区要加宽或缩窄
  • 页面底部导航要改布局
  • 加一种新的页面布局模式
03
components.css
管"零件长什么样"。22类通用组件，按用途分四层。
修改频率：偶尔改
被依赖：level / philosophy / mainline / responsive
依赖：base.css + layout.css
管什么（22类）
  • .tree-*树形目录节点——箭头、图标、子节点竖线
  • .tag-*标签——核心/辅助/实践/加餐/课时等7种
  • .btn-*按钮——默认/主色/危险/幽灵/小号/图标/按钮组
  • .card-*卡片——默认/无内边距/图标卡/两列三列网格
  • .search-box搜索框——默认和大号两种
  • .trap-box教学陷阱警示框
  • .checklist-box检验标准清单（含可交互勾选版和进度条）
  • .alert-*提示框——警告/危险/成功/信息/强调5种
  • .badge徽章/徽标
  • .goal-list教学目标列表——必须/期望
  • .step-list教学步骤列表——带序号圆点
  • .hours-table课时分配表
  • .sim-table手动模拟表格——"画表格"教学工具
  • .code-block代码块——带语言标签和复制按钮
  • .tabs选项卡
  • .timeline-*时间线——竖线、圆点、级别标签、内容
  • .toolbar工具栏
  • blockquote引用块（含铁律引用特殊样式）
  • .empty-state空状态
  • .skeleton骨架屏加载占位
  • .back-to-top返回顶部按钮
  • .progress-*进度条、阅读进度条
不管什么
  • 不管这些组件放在页面哪个位置（layout管）
  • 不管某个级别用什么颜色（level管）
  • 不管理念板块专属组件（philosophy管）
  • 不管主线板块专属组件（mainline管）
  • 不写具体颜色值，只用base的变量
什么时候打开
  • 加一种新的通用组件
  • 调整现有组件的视觉细节（卡片阴影、按钮间距）
  • 修改陷阱框或检验清单的结构
  • 调整代码块的头部样式
04
level.css
管"八个级别各自的颜色"。一个类名切换整个级别上下文。
修改频率：可能微调
被依赖：responsive
依赖：base + layout + components
管什么
  • .level-N变量映射——把 --lv-color 指向 --levelN，加 .level-3 全页自动用L3色
  • .lv-num级别序号圆圈——4种尺寸（xs/sm/默认/lg）、实心/描边
  • .lv-label级别文字标签——"L3 数组期"这种
  • 染色用 --lv-color 覆盖components中定义的组件颜色：侧边栏激活态、核心标签、课时标签、检验清单圆点、陷阱框左边框、步骤列表、代码块头部、选项卡、引用块、阅读进度条
  • .lv-progress级别进度条——显示当前级别完成多少课时
  • .lv-timeline级别时间轴——知识点横向时间线
  • .lv-topic-grid级别概览卡片网格——知识点卡片hover顶部级别色细线
  • .lv-summary级别摘要信息卡——课时/周数/知识点数
  • .lv-difficulty难度指示器——填充方块表示1~8
  • .lv-transition级别过渡提示——"L3到L4能力发生了什么变化"
  • .levels-bar八级横向对比条——首页展示递进关系
不管什么
  • 不定义组件的形状和结构（components管）
  • 不定义颜色值本身（base管 --level1~8）
  • 不管布局（layout管）
  • 不管其他板块的专属样式
核心机制
components.css 里的组件只认 --lv-color，不写死颜色值。

level.css 做 .level-3 { --lv-color: var(--level3); } 这一次映射。

组件不知道自己处于哪个级别，也不需要知道。级别的切换是上层的事。
什么时候打开
  • 新增一种级别专属组件
  • 调整某个级别色在某种组件上的表现
  • 加L9/L10时扩展变量映射
  • 调整级别页面的顶部光晕氛围
05
philosophy.css
管"教学理念板块的专属组件"。只在理念相关页面出现，其他页面不加载。
修改频率：偶尔加新内容
被依赖：responsive
依赖：base + layout + components
管什么（12类）
  • .laws-grid铁律索引卡片——2×2网格，hover红色竖条
  • .law-detail-*铁律详情页——大号序号、一句话高亮块
  • .law-compare"错误做法 vs 正确做法"左右对比块
  • .law-one-liner一句话总结的高亮块
  • .three-pass"裸用→组合→变体"三列可视化
  • .understand-first"先理解后背"流程图（含错误顺序打叉版）
  • .versus-table"考点 vs 能力终点"对照表
  • .issue-card致命编排问题卡片——顶部渐变线
  • .rhythm-*教学节奏——八级方块图、条形图、关键提示
  • .trap-grid陷阱汇总页网格布局和严重度标记
  • .check-level-group检验标准汇总页按级别折叠展示
  • .law-ref教案页中引用铁律的内嵌引用块
不管什么
  • 不重定义 components 已有的 .trap-box / .checklist-box 基础样式
  • 不管其他板块（级别教案、主线、辅助资源）
  • 不写通用组件
什么时候打开
  • 加一条新的铁律详情页
  • 调整对比块的布局
  • 修改教学节奏图的样式
  • 在教案页中新增一种理念引用样式
按需加载：做L3的数组教案页，不加载这个文件。做理念首页或铁律详情页，才加载。不相关的代码不进入页面。
06
mainline.css
管"能力主线专属组件"。只在主线相关页面出现，其他页面不加载。
修改频率：偶尔改
被依赖：responsive
依赖：base + layout + components
管什么（11类）
  • .mainline-grid五条主线总览布局，递归线标注为"最重要"
  • .ml-timeline-*主线详情页纵向时间线——每个节点是卡片，比总览更详细
  • .cross-matrix-table五条主线×八级交叉矩阵
  • .ml-progress单条主线整体完成度——分段进度条（done/current/pending）
  • .ml-deps主线间依赖关系——"数组DP线依赖递归线"
  • .ml-lesson-ref教案页底部"本课属于哪些主线"引用区（含位置指示器）
  • .sidebar-mainlines侧边栏底部"本级别涉及的主线"迷你展示
  • .ml-parallel多条主线并行对比视图
  • .mainline-stats主线详情页统计条——跨级别数、总课时
  • .ml-timeline-bridge时间线节点间的"衔接说明"
  • .timeline-buffer时间线中"缓冲期"特殊标注
不管什么
  • 不重定义 components 已有的 .mainline-card / .timeline-* 基础样式
  • 不管其他板块（理念、级别教案、辅助资源）
  • 不写通用组件
什么时候打开
  • 加一条新的能力主线详情页
  • 调整交叉矩阵的样式
  • 修改主线进度条的视觉
  • 调整教案页底部的主线引用区
按需加载：做L5的贪心教案页，不加载这个文件。做主线总览或递归线详情页，才加载。
07
responsive.css
管"手机上怎么显示"。只做覆盖，不改桌面端，不用 !important。
修改频率：测手机时改
被依赖：无
依赖：前面所有6个文件
管什么
  • @media ≤768px主断点——侧边栏变抽屉、多列变单列、树形缩进收紧、隐藏部分标签
  • @media ≤480px小屏微调——字号再降、间距再收、组件极限压缩
  • @media 769~1024px平板横屏——侧边栏缩窄、网格列数调整
  • .sidebar.open移动端侧边栏抽屉——fixed全屏、transform滑入
  • .sidebar-overlay抽屉遮罩——点击关闭侧边栏
  • @media print打印——隐藏导航/交互元素、侧边栏变内联、去背景色保留结构
  • reduced-motion无障碍——禁用所有动画和过渡
  • high contrast无障碍——加粗边框、提高弱色对比度
不管什么
  • 不定义桌面端样式（只覆盖）
  • 不新增组件（只调整已有组件在手机上的表现）
  • 不用 !important（靠加载顺序保证优先级）
  • 不写新的CSS变量
为什么必须最后加载
CSS层叠规则：后加载的覆盖先加载的。

responsive 里的 @media 只在手机上生效，但它能覆盖前面所有文件的同名选择器。如果它不是最后加载，前面的文件里如果也写了媒体查询，优先级会乱。

写顺序也必须是最后：你要先知道桌面端的最终样式长什么样，才能知道手机上要覆盖什么。
什么时候打开
  • 手机上侧边栏挡了内容
  • 某个表格在手机上溢出
  • 树形目录嵌套太深在手机上看不清
  • 打印出来的页面排版有问题
  • 加了新组件后需要补手机端适配
完整依赖链
base.css 无依赖，所有文件的起点
layout.css 依赖 base 的变量
components.css 依赖 base + layout 的容器
level.css 三者互不依赖，可按需加载 philosophy.css mainline.css
responsive.css 依赖前面所有文件，覆盖它们的手机端样式
