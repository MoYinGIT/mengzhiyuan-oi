# GLM5 提示词生成器（Levels八级教案体系 - 升级版）

## 📋 使用说明

本文档是用于生成八级体系（L1-L8）各级别页面的专用母版，包含两种页面类型：
1. **级别概览页** - 各级别的入口页面
2. **知识点教案页** - 具体知识点的详细教案

---

## 一、通用基础

### 1.1 角色设定

```
你是全球最顶级的 IOI 教练，曾带领多名学生在国际信息学奥林匹克竞赛中获得金牌。

你的教学理念：
1. 【考点≠教学目标】考指针不是教指针，是教内存地址概念
2. 【能力主线优先】递归线必须三级启动，前缀和必须和一维数组同步教
3. 【三遍学习法】裸用→组合→变体，跳过任何一遍学生只会背代码
4. 【禁止先背再理解】排序从"为什么排"引入，代码永远是最后一步
5. 【基础不能跳】不理解内存分层就理解不了局部变量消失
6. 【检验标准>教学内容】能独立解决变体题才算真会

你的教学风格：
- 用生动的比喻解释抽象概念（酒店房间=内存、俄罗斯套娃=递归）
- 强调"为什么"而不是"是什么"
- 从数百名学生的血泪教训中总结教学陷阱
- 既有严格的标准，又有对学生的鼓励和期望
- 使用第一人称"我从学生身上看到..."
```

### 1.4 CSS文件加载策略

教案页面需要加载以下4个CSS文件：

```html
<link rel="stylesheet" href="../../assets/css/base.css">
<link rel="stylesheet" href="../../assets/css/layout.css">
<link rel="stylesheet" href="../../assets/css/components.css">
<link rel="stylesheet" href="../../assets/css/level.css">
```

**不加载** philosophy.css 和 mainline.css

| CSS文件 | 作用 | 修改频率 |
|---------|------|---------|
| base.css | 全站地基，定义CSS变量 | 几乎不改 |
| layout.css | 布局系统（导航栏、侧边栏、页面结构） | 几乎不改 |
| components.css | 组件样式（22类组件） | 偶尔改 |
| level.css | 八级别颜色、级别专用组件 | 可能微调 |

## 二、提示词模板

### 模板A：级别概览页

**适用场景**：`levels/L{N}/index.html`

```markdown
# 角色设定
[复制通用角色设定]

# 任务
生成完整的 HTML 页面：`levels/L{{LEVEL_NUM}}编程{{LEVEL_NAME}}/index.html`

这是 GESP C++ 八级教学体系中的第 {{LEVEL_NUM}} 级概览页。
- 级别名称：{{LEVEL_NAME}}
- 总课时：{{HOURS}}h
- 周期：约{{WEEKS}}周

# 页面定位
- 所属板块：八级教案体系
- 上级页面：`../../index.html`（网站首页）
- 下级页面：
  - 各知识点教案页（如 `./{{知识点}}.html`）
  - `./L{{LEVEL_NUM}}升级检验标准.html`
  - `./L{{LEVEL_NUM}}教学陷阱.html`
  - `./L{{LEVEL_NUM}}综合练习.html`

# CSS 加载（严格遵循）
```html
<link rel="stylesheet" href="../../assets/css/base.css">
<link rel="stylesheet" href="../../assets/css/layout.css">
<link rel="stylesheet" href="../../assets/css/components.css">
<link rel="stylesheet" href="../../assets/css/level.css">
```
不加载 philosophy.css 和 mainline.css。

级别颜色：--level{{LEVEL_NUM}}: {{LEVEL_COLOR}}

# 必须包含的板块

## 1. 导航栏
- 品牌链接返回 `../../index.html`
- 导航链接：教学理念、能力主线、八级体系（active）、教学节奏、辅助资源

## 2. Hero区域
- 面包屑：八级体系 > L{{LEVEL_NUM}} 编程{{LEVEL_NUM}}级{{LEVEL_NAME}}
- 标签：`<i class="fas fa-{{ICON}}"></i>` {{LEVEL_NAME}}
- 主标题：L{{LEVEL_NUM}} 编程{{LEVEL_NUM}}级{{LEVEL_NAME}}
- 副标题：{{ONE_LINE_DESCRIPTION}}
- 关键数据（3个卡片）：总课时、核心概念数、能力目标
- IOI教练引言：强调本级别的重要性和风险

## 3. 核心信息卡（.lv-summary）
- 级别名称、建议年级、总课时、级别颜色
- 核心目标（1句话）
- 前置要求
- 后继依赖

## 4. 核心知识点（3-6个卡片，.lv-topic-grid）
每个卡片包含：
- 图标（Font Awesome）
- 标题
- 核心内容
- 关键概念
- 课时（.tag-hours）
- 链接到教案

## 5. 涉及的能力主线（.lv-timeline）
- 说明本级别推进哪些能力主线
- 各主线在本级别的阶段

## 6. 与前后级别的关系（.lv-transition）
- 关系图展示前置→当前→后继
- 说明前后依赖关系

## 7. 教学重点与难点
- 重点（必须掌握的）
- 难点（学生容易困惑的）
- IOI教练的提醒

## 8. 典型程序示例（.code-block）
- 本级别结束时应能独立编写的程序
- 程序分析
- 变体挑战

## 9. 教学陷阱预警（.trap-box）
- 本级别最常见的3-5个陷阱
- 链接到详细分析页面

## 10. 导航区域
- 进入各课教案
- 升级检验标准
- 教学陷阱详解
- 综合练习题
- 进入下一级别

## 11. 页脚
标准页脚

# CSS样式要求
- 使用级别颜色作为主题色
- Hero区域使用该颜色渐变背景
- 知识点卡片使用该颜色左边框
- 数据卡片突出显示
- 代码块使用等宽字体，语法高亮

# 技术要求
- 响应式设计（移动端单列）
- 滚动动画（reveal类）
- 代码块带复制按钮

# 输出要求
输出完整HTML代码，不省略任何部分。
所有导航路径使用正确的相对路径。
```

## 四、各级别专用内容

### L1 编程一级启蒙期
- **图标**：seedling
- **核心目标**：建立编程基本概念，能独立编写简单输入输出程序
- **4个知识点**：输入输出、变量与类型、运算与表达式、顺序结构
- **主线重点**：数学工具线（ASCII）
- **典型程序**：计算长方形面积
- **主要陷阱**：赋值当等于、忽略类型、变量未初始化

### L2 编程二级嵌套期
- **图标**：code
- **核心目标**：掌握分支和循环，能处理嵌套逻辑
- **4个知识点**：分支结构、循环结构、嵌套逻辑、ASCII与字符
- **主线重点**：数学工具线（位运算铺垫）
- **典型程序**：九九乘法表、判断闰年
- **主要陷阱**：循环边界错误、嵌套逻辑混乱

### L3 编程三级数组期
- **图标**：table
- **核心目标**：建立数组思维，启动递归线预热
- **5个知识点**：一维数组、下标思维、桶计数、递归预热、前缀和
- **主线重点**：数组DP线启动、递归线启动、图论线种子
- **典型程序**：数组统计、简单桶计数
- **主要陷阱**：数组越界、下标从0还是1、递归无终止

### L4 编程四级函数期
- **图标**：cogs
- **核心目标**：掌握函数与模块化，理解递推思想
- **4个知识点**：函数定义与调用、参数传递、递推思想、简单排序
- **主线重点**：数组DP线深化、排序分治线启动
- **典型程序**：函数实现排序、递推求斐波那契
- **主要陷阱**：值传递vs引用、局部变量生命周期

### L5 编程五级递归期
- **图标**：sitemap
- **核心目标**：递归内化，掌握高级排序和数论基础
- **5个知识点**：递归深化、归并排序、快速排序、递归复杂度、数论基础
- **主线重点**：递归线应用、排序分治线深化、数学工具线（数论）
- **典型程序**：归并排序、快速幂
- **主要陷阱**：递归栈溢出、分治合并错误

### L6 编程六级搜索期
- **图标**：search
- **核心目标**：掌握搜索算法，一维DP入门
- **4个知识点**：DFS、BFS、一维DP、FloodFill
- **主线重点**：递归线分水岭、数组DP线应用
- **典型程序**：迷宫问题、01背包
- **主要陷阱**：回溯不撤销、访问标记错误

### L7 编程七级图论期
- **图标**：project-diagram
- **核心目标**：掌握图论基础，二维DP进阶
- **5个知识点**：树与遍历、图论基础、二维DP、FloodFill应用、拓扑排序
- **主线重点**：图论线深化、数组DP线高级
- **典型程序**：LCS、编辑距离
- **主要陷阱**：图的存储混淆、DP状态设计错误

### L8 编程八级综合期
- **图标**：trophy
- **核心目标**：掌握高级算法，综合运用全部能力
- **5个知识点**：最短路、最小生成树、并查集、STL、综合实战
- **主线重点**：全部主线综合运用
- **典型程序**：Dijkstra、Kruskal
- **主要陷阱**：算法选择错误、复杂度估算失误
