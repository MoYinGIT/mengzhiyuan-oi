# 审查红线（绝对不可违背的规则）
你必须对照以下5条红线进行扫描，任何一条不满足都必须标记为【致命错误】并强制修复：

## 红线一：零ASCII Art容忍度
- **规则**：全站任何HTML文件中，严禁出现 `<pre>` 包含框线字符（如 `┌─┐`, `+--+`, `│`, `->`）绘制的图表。
- **修复标准**：必须替换为 `data-type="memory"`（内存布局）、`data-type="tracker"`（变量跟踪表）或 `data-type="mermaid"`（流程图）组件。

## 红线二：CSS架构的单向依赖与按需加载
- **规则**：CSS文件必须遵循 `base <- layout <- components <- level/philosophy/mainline <- responsive` 的单向依赖。
- **致命错误示例**：在 `levels/L1/lessons/xxx.html` 中引入了 `philosophy.css` 或 `mainline.css`（这叫冗余加载）。
- **修复标准**：教案页只能引入 `base, layout, components, level, responsive, teaching-visuals`。

## 红线三：IOI教练视角的内容缺失
- **规则**：每个课时HTML（`L*-**-*.html`）必须包含三大教学组件：
  1. `trap-box`（教学陷阱预警，不能只是文字，必须有特定class）
  2. `checklist-box`（自查清单，必须可量化，如“能独立写出xxx”）
  3. IOI底层思维（不能只教语法，必须解释“为什么”，如变量是“内存地址的别名”而非“盒子”）
- **致命错误示例**：直接给排序代码而不讲“为什么要排”；讲指针而不画内存图。

## 红线四：数据驱动的可视化组件规范
- **规则**：所有 `data-type` 组件的 `data-*` 属性中的JSON必须是**严格合法**的格式。
- **致命错误示例**：JSON数组或对象最后多了一个逗号（尾随逗号会导致JS解析崩溃）。
- **修复标准**：剔除所有尾随逗号，确保双引号转义正确。

## 红线五：CSS变量的绝对统治
- **规则**：在 `components.css`, `level.css`, `philosophy.css`, `mainline.css` 中，**绝对不允许出现硬编码的颜色值**（如 `#fff`, `color: red`）。
- **修复标准**：必须替换为 `var(--fg)`, `var(--level1)` 等在 `base.css` 中定义的变量。
